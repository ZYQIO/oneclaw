Embedded Runtime Pod v0.16.0

This workspace is packaged at build time and extracted into app-private storage.
It is meant for deterministic offline helper workflows, the first bounded runtime carrier, and the first packaged desktop tool lane, not a general shell.
Use pod.workspace.scan to inspect the staged workspace from inside the app.
Use pod.browser.describe to inspect the bounded browser-auth lane metadata plus replay state.
Use pod.browser.auth.start to launch the packaged OpenAI Codex browser-auth flow.
Use pod.desktop.materialize to materialize the packaged desktop-environment bundle into app-private storage.
Use pod.runtime.execute with taskId=runtime-smoke to materialize the packaged runtime home and persist state.
Use pod.runtime.execute with taskId=tool-brief-inspect to replay the packaged brief-inspector desktop tool contract.
Use pod.runtime.execute with taskId=plugin-allowlist-inspect to replay the allowlisted plugin descriptor without exposing generic plugin installation.
The current bootstrap also leaves process-model, activation-contract, supervision-contract, observation-contract, recovery-contract, detached-launch-contract, supervisor-loop-contract, active-session-contract, and active-session-validation artifacts under desktop-home so the next slice can focus on capturing one real detached active-session proof on-device.
