# UI 设计规范

## 文档信息
- **创建日期**: 2026-02-27
- **最后更新**: 2026-02-27
- **状态**: 草稿
- **参考风格**: Google Gemini App（Material 3，简洁，宽敞）

## 设计原则

1. **简洁宽敞**：充裕的留白，无视觉杂乱。内容占据核心位置。
2. **极简装饰**：减少边框、分隔线和装饰带来的视觉噪音。用间距和微妙的背景创造层次。
3. **温暖亲切**：金色/琥珀色强调色传递温暖感。整体感觉友好而非冰冷。
4. **一致的交互模式**：所有页面使用相同的手势和模式，不给用户意外。
5. **内容优先**：UI 元素服务于内容（对话、Agent 配置等），而非反过来。

## 颜色与主题

应用使用暖金色/琥珀色主色的 Material 3 主题。主题文件从 Material Theme Builder 导出，存储在 `docs/design/material-theme/` 中。

- **主色**: `#6D5E0F`（暖金色/琥珀色）
- **Tertiary**: 绿色强调色
- **变体**: 亮色、暗色、中对比度、高对比度
- **动态颜色**: Android 12+ 上启用（跟随用户壁纸）；旧设备回退到定义的配色方案

详见 `docs/design/material-theme/ui/theme/Color.kt` 的完整颜色定义。

## 排版

- **字体族**: Roboto（通过 Google Fonts provider）
- **体系**: 标准 Material 3 排版体系
- Display、Headline、Title、Body、Label 各层级按 Material 3 定义

详见 `docs/design/material-theme/ui/theme/Type.kt` 的定义。

## 形状

- **圆角半径**: Material 3 默认值
  - 小组件（chip、小卡片）: 8dp 圆角
  - 中等组件（卡片、对话框）: 12dp 圆角
  - 大组件（sheet、大卡片）: 16dp 圆角
  - 超大（输入框 pill 形状）: 28dp 圆角

## 整体应用结构

### 导航模型

```
+-----------------------------------------------+
|  [汉堡菜单]  OneClawShadow          [设置]     |  <- 顶部应用栏
|-----------------------------------------------|
|                                               |
|              主内容区域                         |
|         （聊天 / 设置 / 等）                    |
|                                               |
+-----------------------------------------------+
```

- **左上**: 汉堡菜单图标 -- 从左侧打开导航抽屉
- **右上**: 设置齿轮图标 -- 导航到设置页面
- **无底部导航栏**: 通过抽屉进行顶级导航
- **导航抽屉**: 仅包含会话列表和新建对话按钮

### 导航抽屉（会话列表）

抽屉是访问和管理会话的主要方式。

```
+----------------------------------+
| +------------------------------+ |
| |  [+]  新建对话               | |  <- 新建对话按钮（顶部）
| +------------------------------+ |
|                                  |
|  今天                            |  <- 日期分组标题（可选）
|  +----------------------------+  |
|  | 会话标题                   |  |
|  | 最后消息预览...             |  |  <- 弱化的次要文字
|  | 2 分钟前         [Agent]   |  |  <- 时间 + Agent 标记
|  +----------------------------+  |
|  +----------------------------+  |
|  | 另一个会话                 |  |
|  | 预览文字在这里...           |  |
|  | 昨天             [Agent]   |  |
|  +----------------------------+  |
|  +----------------------------+  |
|  | 更早的会话                 |  |
|  | ...                        |  |
|  | 2月20日           [Agent]  |  |
|  +----------------------------+  |
|                                  |
|  （向左滑动删除项目）             |
|                                  |
+----------------------------------+
```

**会话列表项内容：**
- **标题**: 主文本，单行，省略号溢出
- **最后消息预览**: 次要文本，1-2 行，弱化颜色
- **时间戳**: 相对时间（"2 分钟前"、"昨天"、"2月20日"）
- **Agent 标记**: 小 chip/标签显示当前 Agent 名称

**会话列表行为：**
- 按最后活跃时间排序（最近的在前）
- 向左滑动删除项目（带 Snackbar 撤销，约 5 秒）
- 长按进入选择模式进行批量删除
- 点击打开/恢复该会话
- 顶部"新建对话"按钮创建新聊天

**延迟会话创建**: 点击"新建对话"关闭抽屉并显示空聊天界面。在用户发送第一条消息之前不会在数据库中创建会话。

## 页面规格

### 1. 聊天页面（主页面）

这是用户看到的主要页面，即聊天对话界面。

#### 布局结构

```
+-----------------------------------------------+
|  [汉堡菜单] [通用助手 v]            [设置]      |  <- 顶部应用栏
|-----------------------------------------------|
|                                               |
|                                               |
|           （空状态或消息）                       |
|                                               |
|                                               |
|                                               |
|                                               |
|  +---+                                        |
|  |   | AI 响应文本在这里。这是左对齐              |
|  |图标| 无气泡背景的文本。可以是多行              |
|  +---+ Markdown 渲染的内容。                    |
|                                               |
|        [复制] [重新生成] [分享]                  |  <- 操作行
|                                               |
|                        +--------------------+ |
|                        | 用户消息文本        | |  <- 右对齐气泡
|                        +--------------------+ |
|                                               |
|-----------------------------------------------|
|  +---------------------------------------+    |
|  | [+]  消息...                     [=>] |    |  <- 输入框（pill 形状）
|  +---------------------------------------+    |
+-----------------------------------------------+
```

#### 顶部应用栏
- **左侧**: 汉堡菜单图标（打开抽屉）
- **中间**: 当前 Agent 名称带下拉箭头 -- 点击打开 Agent 选择器（底部弹出或下拉菜单）
- **右侧**: 设置齿轮图标

#### 空状态（新对话）
没有消息时：

```
+-----------------------------------------------+
|  [汉堡菜单] [通用助手 v]            [设置]      |
|-----------------------------------------------|
|                                               |
|                                               |
|                                               |
|                                               |
|          有什么我可以帮你的吗？                   |  <- 居中问候语
|                                               |
|                                               |
|                                               |
|                                               |
|-----------------------------------------------|
|  +---------------------------------------+    |
|  | [+]  消息...                     [=>] |    |
|  +---------------------------------------+    |
+-----------------------------------------------+
```

- 问候文字："有什么我可以帮你的吗？"（或对应的本地化版本）
- 在消息区域垂直居中
- 弱化文字颜色（Material 3 `onSurfaceVariant`）
- 无 logo、无插图、无建议 chip -- 仅简单的问候语

#### 消息展示

**用户消息：**
- 右对齐
- 背景: `primaryContainer` 颜色，圆角（18dp）
- 文字颜色: `onPrimaryContainer`
- 内边距: 水平 12dp，垂直 8dp
- 最大宽度: 约屏幕宽度的 80%
- 无头像/图标

**AI 响应：**
- 左对齐，全宽（无气泡，无背景）
- 左侧小 AI 图标（16dp，与文本第一行对齐）
- 文本从图标右侧开始
- 文字颜色: `onSurface`
- 支持完整 Markdown 渲染（粗体、斜体、列表、标题、行内代码、代码块）
- 无背景色 -- 通过对齐方式和无气泡来与用户消息视觉区分

**AI 响应操作行：**
- 出现在每条完成的 AI 响应下方
- 一行小图标按钮，左对齐在响应文本下方
- 图标: 复制、重新生成、分享
- 图标大小: 20dp，弱化颜色（`onSurfaceVariant`），图标间距 8dp
- 重新生成和分享为 V1 可选项（工作量大则记录为未来功能）
- 点击复制复制原始文本（Markdown 源码，非渲染后的内容）

**消息间距：**
- 不同发送者的消息间垂直间距 16dp
- 同一发送者的连续消息间距减小: 4dp

#### 思考块

附在 AI 响应消息上，显示在主要响应文本上方。

```
+-------------------------------------------+
| [>] 正在思考...                            |  <- 折叠状态
+-------------------------------------------+
  AI 图标  实际的响应文本在思考块之后
           显示在这里。
           [复制] [重新生成] [分享]
```

```
+-------------------------------------------+
| [v] 思考                                   |  <- 展开状态
|                                           |
|   模型的内部推理文本以较浅/弱化的             |
|   样式和微妙的背景显示在这里...               |
|                                           |
+-------------------------------------------+
  AI 图标  实际的响应文本在思考块之后
           显示在这里。
           [复制] [重新生成] [分享]
```

- **折叠（默认）**: 单行，箭头图标和"正在思考..."标签
- **展开**: 完整思考文本在带微妙 `surfaceVariant` 背景的卡片中
- **文字样式**: `bodySmall`，颜色 `onSurfaceVariant`（比主响应浅）
- 点击标题行切换展开/折叠

#### 工具调用展示

工具调用内联出现在对话流中，在 AI 响应之间或之内。

**简洁模式（默认）：**
```
  [工具图标] 正在调用 read_file... 完成      <- 单行
```
- 工具图标（16dp）+ 工具名 + 状态
- 状态: "正在调用..."（进行中）或"完成"（已完成）或"失败"（错误）
- 点击展开查看详情（输入/输出）
- 微妙的 `surfaceVariant` 背景，圆角（8dp）

**详细模式：**
```
+-------------------------------------------+
| [工具图标] read_file                       |
|                                           |
|  输入:                                     |
|    { "path": "/storage/notes.txt" }       |
|                                           |
|  结果:                                     |
|    "文件内容..."                            |
|                                           |
|  耗时: 120ms                               |
+-------------------------------------------+
```
- 带 `surfaceVariant` 背景的卡片
- 工具名作为标题
- 输入和结果区域，各自可折叠
- JSON 内容使用代码样式渲染（等宽字体）
- 底部显示耗时

**模式切换：**
- 顶部应用栏中的切换按钮（或溢出菜单）：全局切换简洁/详细模式
- 偏好设置持久化到设置中

**工具调用状态指示器：**
- 等待/执行中: 工具名旁的微妙圆形进度指示器
- 成功: 勾选图标或"完成"文字
- 错误: `error` 颜色的错误图标 + 错误消息
- 超时: 时钟图标 + "已超时" 文字

#### 流式指示器

等待 AI 第一个 token 时：
```
  AI 图标  [... 脉冲点 ...]
```
- 三个脉冲点动画，左对齐在 AI 响应将出现的位置
- 第一个 token 到达后被实际文本替换

流式传输中：
- 文本逐步渲染，逐字符（或逐块）
- 流式文本末尾有光标/闪烁线指示器
- 自动滚动跟随新内容

#### 输入区域

```
+-----------------------------------------------+
|  +---------------------------------------+    |
|  | [+]  输入消息...                 [=>] |    |  <- 输入框
|  +---------------------------------------+    |
+-----------------------------------------------+
```

Agent 选择器已移至顶部应用栏（见上方）。输入区域简洁，专注于消息编写。

**输入框：**
- Pill 形状（超大圆角，28dp 半径）
- 背景: `surfaceContainerHigh`（比屏幕背景略高）
- 左侧: 附件/加号按钮（V1 中可隐藏，用于未来扩展）
- 中间: 文字输入，提示文字"输入消息..."
- 右侧: 发送按钮
  - 启用: `primary` 颜色，填充圆形图标
  - 禁用: `onSurfaceVariant` 降低透明度（输入为空或请求进行中时）
- 生成过程中: 发送按钮变为停止按钮（`error` 颜色的方形图标）
- 多行扩展: 输入框垂直增长最多约 4 行，之后内部滚动

**Agent 选择（点击顶栏 Agent 名称时）：**

```
+-----------------------------------------------+
|         选择 Agent                             |
|-----------------------------------------------|
|  [勾选] 通用助手                     内置      |
|          写作助手                              |
|          代码助手                              |
|          数据分析师                             |
+-----------------------------------------------+
```

- 底部弹出或弹出菜单
- 显示所有可用 Agent
- 当前 Agent 有勾选标记
- 内置 Agent 有微妙标记
- 点击 Agent 立即切换
- 聊天中出现系统消息："已切换到 [Agent 名称]"

#### Agent 切换指示器（聊天中）

对话中途切换 Agent 时出现系统消息：

```
        --- 已切换到 写作助手 ---
```

- 居中，全宽
- 弱化文字颜色（`onSurfaceVariant`）
- 小字体（`labelMedium`）
- 文字两侧有细水平线（分隔线样式）
- 这是持久化在数据库中的 SYSTEM 消息类型

#### 错误消息

错误内联出现在对话中：

```
  [!] 无法连接。请检查您的网络并重试。
      [重试]
```

- 左侧错误图标（`error` 颜色）
- 错误文字颜色 `onErrorContainer`
- 背景: `errorContainer` 带圆角
- 可选的"重试"文字按钮

#### 滚动行为
- 新消息和流式传输时自动滚动到底部
- 流式传输期间用户向上滚动时，自动滚动暂停
- 滚动离开底部时出现"滚动到底部"FAB：
  ```
                                    [v]   <- 小圆形 FAB
  ```
  - 带下箭头图标的小圆形按钮
  - 位于右下角，输入区域上方
  - 点击滚动到最新消息

从设置 > 管理 Provider 进入。

```
+-----------------------------------------------+

|-----------------------------------------------|
|                                               |
|  +-------------------------------------------+|
|  | [O]  OpenAI                               ||
|  |      3 个模型         已连接              ||
|  +-------------------------------------------+|
|  | [A]  Anthropic                            ||
|  |      2 个模型         已连接              ||
|  +-------------------------------------------+|
|  | [G]  Google Gemini                        ||
|  |      未配置                                ||
|  +-------------------------------------------+|
|                                               |
|  自定义                                        |
|  +-------------------------------------------+|
|  | 我的本地服务器                              ||
|  |      1 个模型         已连接              ||
|  +-------------------------------------------+|
|                                               |
+-----------------------------------------------+
```

- **[+] 按钮**: 右上角操作按钮，添加自定义 provider
- **预配置 provider**: 始终显示（OpenAI、Anthropic、Gemini），带状态
- **自定义 provider**: 列在下方的独立区域中

### 4. Provider 详情页面

从 Provider 列表中点击某个 provider 进入。

```
+-----------------------------------------------+
|  [<-]        OpenAI                           |
|-----------------------------------------------|
|                                               |
|  API Key                                      |
|  +-------------------------------------------+|
|  | sk-...abc1234                   [眼睛]    ||
|  +-------------------------------------------+|
|  [测试连接]                                    |
|                                               |
|  连接: 已连接 (2 分钟前检查)                    |  <- 状态文字
|                                               |
|  可用模型                                      |
|  +-------------------------------------------+|
|  | (*) gpt-4o              动态       [星标] ||  <- 默认模型
|  | ( ) gpt-4o-mini         动态              ||
|  | ( ) o1                  动态              ||
|  | ( ) o3-mini             动态              ||
|  +-------------------------------------------+|
|  [刷新模型]                                    |
|                                               |
|  +-------------------------------------------+|
|  | 启用                          [开关]      ||
|  +-------------------------------------------+|
|                                               |
+-----------------------------------------------+
```

- **API key 字段**: 默认遮掩。眼睛图标切换可见性。点击可编辑。
- **测试连接按钮**: 描边按钮。测试期间显示加载旋转，然后在下方显示结果文字。
- **模型列表**: 单选按钮或星标图标用于选择默认模型。每个模型显示其来源标签（动态/预设/手动）。
- **刷新模型按钮**: 文字按钮，触发重新获取模型列表。
- **启用开关**: 启用/停用此 provider。
- 预配置 provider: 名称和 URL 只读。自定义 provider: 所有字段可编辑。
- **删除按钮**: 底部（仅自定义 provider），需确认对话框。

```
|  +-------------------------------------------+|
|                                               |
+-----------------------------------------------+
```

- **内置 Agent**: 所有字段只读。仅显示"克隆"按钮（无删除、无保存）。
- **自定义 Agent**: 所有字段可编辑。显示保存、克隆和删除按钮。
- **System prompt 字段**: 大文本区域，可扩展
- **工具选择**: 所有可用工具的复选框列表
- **首选模型**: 点击打开选择器（显示所有活跃 provider 的所有模型）。"未设置"表示使用全局默认。
- **保存按钮**: 在应用栏右上角。未做更改前禁用。

用于会话删除的撤销。

- 标准 Material 3 AlertDialog
- 破坏性操作按钮使用 `error` 颜色
- 取消按钮为文字按钮

### 加载状态

- **全屏加载**: 居中的圆形进度指示器（很少使用，仅初始数据加载）
- **内联加载**: 触发元素旁的小圆形指示器（例如"测试连接"按钮旁）
- **流式加载**: 消息区域的脉冲点（见上方流式指示器）

- 使用 Snackbar 作为临时状态消息（保存确认等）
- 持续时间: 确认消息短（约 3 秒），带撤销的操作长（约 5 秒）

- 停止生成: 中等触

## 深色模式

应用支持三种主题模式：
1. **亮色**: 浅色背景，深色文字
2. **深色**: 深色背景，浅色文字
3. **跟随系统**: 跟随 Android 系统设置

Material 3 主题文件已包含亮色和深色两种配色方案。Android 12+ 上的动态颜色自动适配两种模式。

深色模式关键考虑：
- 用户消息气泡: 使用 `primaryContainer`（自动适配）
- AI 响应文字: 使用 `onSurface`（自动适配）
- 工具调用卡片: 使用 `surfaceVariant`（自动适配）
- 错误消息: 使用 `errorContainer`（自动适配）
- 如果正确应用 Material 3 主题，无需自定义深色模式覆盖

## 设计决策日志

| 决策 | 理由 |
|------|------|
| 抽屉放会话列表（非独立首页） | 匹配 Gemini app 模式。聊天是主要体验。 |
| AI 响应无气泡 | 更简洁，匹配 Gemini。通过对齐方式 + 图标进行视觉区分。 |
| Agent 选择器在顶栏中间 | 取代静态 Agent 名称；避免顶栏和输入区域重复显示；保持输入区域简洁。 |
| 设置在右侧，抽屉在左侧 | 清晰的空间分区。匹配常见 Android 模式。 |
| 空状态用简单文字问候 | 极简，不压迫用户。无建议 chip（与 Agent 相关）。 |
| 延迟会话创建 | 避免列表中的空会话。会话仅在有内容时存在。 |
| Agent 切换用系统消息 | 保留切换历史。对话中清晰的视觉断点。 |
| 重新生成/分享为 V1 可选 | 核心价值在聊天 + 工具调用。这些是打磨功能。 |

## 与其他文档的关系

- **PRD 文档** 定义每个功能做什么（功能需求）
- **本 UI 设计规范** 定义应用的外观和体验（视觉和交互设计）
- **RFC 文档** 定义每个功能如何构建（技术实现）
- RFC 应引用本规范获取 UI 相关的实现细节
