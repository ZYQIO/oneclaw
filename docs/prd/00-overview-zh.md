## 产品愿景

OneClawShadow 是一个运行在 Android 上的移动端 AI Agent 运行环境。用户通过聊天界面与 AI 模型交互，AI 可以自主调用工具来完成任务。核心理念是用户的手机本身就是 Agent 的执行环境——不需要搭建服务器，也不依赖任何后端服务。

该应用在设计上**零后端依赖**：用户自带 API key、自带云存储账号，所有数据完全由用户掌控。

OneClawShadow 不是编程工具或 IDE，它是一个运行在手机上的通用 AI Agent。

## 目标用户

### 主要用户群体
1. **技术型用户**：希望手机上的 AI 不仅能聊天，还能自主执行操作（读取文件、调用 API、获取数据）。
2. **重视数据所有权的用户**：希望使用自己的 API key，数据存储在本地，不依赖任何第三方后端。
3. **高级用户**：希望通过自定义 Agent（不同的 system prompt 和工具配置）来定制 AI 行为。

## 核心价值主张

1. **手机上的 AI Agent** -- 无需搭建服务器，手机就是 Agent 运行环境
2. **工具增强的 AI** -- AI 不仅能聊天，还能通过设计良好、可扩展的工具系统执行操作
3. **用户完全掌控** -- 用户拥有自己的 API key、数据和 Agent 配置
4. **可扩展架构** -- 清晰的工具接口，可以随时添加新能力

## 产品范围

### 包含在 V1 中
- 基于聊天的 AI 模型交互（仅文字）
- 流式响应展示
- 工具调用执行与可视化（简洁/详细两种模式，用户可切换）
- Agent 管理（创建/编辑 Agent，自定义 system prompt 和工具集）
- 模型/Provider 管理（多 provider 支持、API key 管理、模型选择）
- 可扩展的工具系统，接口定义良好
- 内置基础工具（本地文件读写、获取当前时间、HTTP 请求）
- 会话管理（平铺列表）
- Token 用量追踪与费用估算
- 本地数据存储
- Google Drive 同步
- 推送通知（Agent 任务完成时通知）
- 离线可用（可查看历史记录，应用可用但无法发送请求）

### 不包含在 V1 中
- 多模态输入（图片、语音）-- 架构上会预留扩展能力
- 会话组织（文件夹、标签、分组）
- Agent 配置分享/导入/导出
- 其他云同步方案（Dropbox 等）
- 本地备份导出（zip 打包）
- SSH 连接或远程服务器终端访问
- 任何由我们运营的后端服务
- 用户认证系统（无账号、无登录）

### 模块说明

#### FEAT-001: 对话交互
核心聊天界面。用户输入文字消息发送给 AI 模型，模型的响应以流式方式展示。当模型发起工具调用时，执行过程会在聊天中展示。用户可以在简洁视图（"正在调用工具 X..."）和详细视图（工具名称、参数、返回结果，可展开/折叠）之间切换。

#### FEAT-002: Agent 管理
用户可以创建、编辑和删除 Agent。每个 Agent 包含：
- 名称和描述
- 自定义 system prompt
- 配置的可用工具集（该 Agent 可以使用哪些工具）
- 选定的模型/provider

应用内置几个默认 Agent，用户也可以创建自定义 Agent。

#### FEAT-003: 模型/Provider 管理
用户配置自己的 API provider 和密钥：
- 支持多个 provider（OpenAI、Anthropic、自定义 API endpoint 等）
- API key 安全存储在本地
- 可按 Agent 或按会话选择使用哪个模型
- 管理和测试 API 连接

#### FEAT-004: 工具系统
可扩展的工具执行框架。这是架构上最重要的模块：
- 定义良好的工具注册、调用和结果回传接口/协议
- 工具执行引擎，处理 模型 -> 工具调用 -> 结果 -> 模型 的循环
- 内置基础工具：本地文件读写、当前时间、HTTP 请求
- 每个工具声明所需的 Android 权限（如有）
- 当工具需要 Android 系统权限（位置、存储等）时，弹出标准的 Android 权限请求对话框
- 工具默认自动执行，无需用户确认
- 面向可扩展性设计——添加新工具应该简单直接

#### FEAT-005: 会话管理
管理多个对话会话：
- 所有会话以平铺列表展示
- 每个会话关联一个 Agent
- 会话历史持久化存储在本地
- 用户可以恢复、删除或查看过去的会话
- 离线状态下可访问会话

#### FEAT-006: Token/费用追踪
由于用户使用自己的 API key，用量可见性非常重要：
- 显示每条消息的 token 数量（输入/输出 token）
- 显示每个会话的 token 用量
- 累计使用统计（按日、按周、按月、总计）
- 基于模型定价的费用估算

#### FEAT-007: 数据存储与同步
所有数据优先存储在本地：
- 会话、Agent 配置、provider 设置存储在本地数据库
- Google Drive 同步，用于跨设备数据备份/恢复
- 同步可由用户手动触发或配置为自动同步
- 用户提供自己的 Google 账号进行同步

#### FEAT-008: 通知
Android 推送通知：
- 当长时间运行的 Agent 任务完成时发送通知
- 点击通知可导航回相关会话

#### FEAT-009: 设置
通用应用配置：
- UI 偏好（主题、字体大小等）
- 默认模型/provider 选择
- 通知偏好
- 数据管理（清除缓存、管理存储空间）

#### FEAT-024: 消息桥接
将外部消息平台连接到 AI Agent 运行时。用户可以通过 Telegram、Discord、Slack、Matrix、LINE 或本地 WebChat 界面与 Agent 交互，无需打开应用。桥接作为前台服务运行，接收来自外部平台的消息，路由到 Agent 处理，并将响应发送回去。支持图片消息、按渠道的访问控制，以及专用的设置页面用于配置和状态监控。作为独立的 `:bridge` Gradle 模块实现。

## 非功能性需求

### 性能要求
- 应用启动时间 < 2 秒
- UI 响应时间 < 100ms
- 流式响应渲染延迟尽可能低
- 工具执行不应阻塞 UI

### 兼容性要求
- Android 版本：8.0（API 26）及以上
- 支持的屏幕尺寸：手机（平板为可选支持）
- 语言支持：英文（主要）、中文

### 安全要求
- API key 在设备上加密存储
- 所有 API 通信通过 HTTPS
- 不向任何我们运营的服务器发送数据
- 本地数据存储使用 Android 标准安全机制

### 可用性要求
- 应用崩溃率 < 0.1%
- 离线时优雅降级（可查看历史、编辑设置、管理 Agent）
- API 调用失败时提供清晰的错误状态

## 依赖与假设

### 外部依赖
- AI 模型 API provider（OpenAI、Anthropic 等）-- 由用户提供
- Google Drive API -- 用于数据同步
- Android 系统 API -- 用于权限、通知、文件访问

### 假设
- 用户拥有至少一个 AI 模型 provider 的 API key
- 如果用户希望使用同步功能，需要有 Google 账号
- 用户了解 AI Agent 和工具调用的基本概念

## 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| API provider 限流或宕机 | 中 | 中 | 清晰的错误提示，重试逻辑 |
| 手机端工具执行能力有限（相比服务器） | 中 | 高 | 聚焦在移动端合理的工具上，可扩展架构 |
| 设备上的 API key 安全性 | 高 | 低 | 使用 Android Keystore 加密，遵循安全最佳实践 |
| 大量对话历史影响性能 | 中 | 中 | 分页加载、懒加载、定期清理选项 |
| Google Drive API 变更 | 低 | 低 | 抽象同步接口，便于切换 provider |

## 架构原则

以下高层原则应指导技术设计（详见 RFC）：

1. **工具接口优先**：工具系统的接口/协议是最关键的架构决策，必须清晰、有版本控制、可扩展。
2. **本地优先**：所有数据存储在本地，同步是附加功能，不是必需品。
3. **Provider 无关**：模型交互层应抽象化 provider 之间的差异。
4. **离线友好**：除了发送 API 请求外，应用应在离线状态下完全可用。
5. **可扩展性优于完整性**：宁可工具少但接口出色，也不要工具多但系统僵化。
