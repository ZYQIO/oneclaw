## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 浏览、预览和管理 AI Agent 在应用内保存的文件，
**以便** 我能够查看生成的内容、将文件分享给其他应用，并保持文件存储的整洁有序。

### 典型场景

1. 用户要求 AI "编写一个对列表进行排序的 Python 脚本"——AI 使用 `write_file` 保存该脚本。用户打开文件浏览器，查看、复制或分享已保存的脚本。
2. 用户要求 AI "保存我们对话的摘要"——文件保存在应用中。用户随后打开文件浏览器，找到并阅读该摘要。
3. 用户发现应用的文件存储占用较大。他们打开文件浏览器，找出不需要的文件并将其删除。
4. 用户想通过即时通讯应用将 AI 生成的文件发送给同事。他们打开文件浏览器，找到该文件，并使用分享操作。
5. 用户在文件浏览器中通过目录层级浏览，查找按类别或日期组织的文件。

## 功能描述

### 概述

文件浏览功能为应用的内部存储提供了一个内置的文件管理器。当 AI Agent 使用 `write_file` 工具保存文件时，这些文件会存储在应用私有存储下的专用目录（`filesDir/user_files/`）中。文件浏览器允许用户浏览该目录结构、预览文件内容、删除不需要的文件，以及通过 Android 分享菜单将文件分享给其他应用。

### 页面区域

#### 1. 文件列表视图

主界面展示当前路径下的文件和目录：
- **面包屑导航**：显示当前路径，各路径段可点击以向上导航
- **目录条目**：以文件夹图标展示，可点击进入
- **文件条目**：以对应类型的图标展示，显示名称、大小和最后修改日期
- **排序**：默认按字母顺序排序，目录排在文件前面
- **空状态**：目录为空时显示"暂无文件"提示

#### 2. 文件预览

点击文件打开预览界面：
- **文本文件**（.txt、.md、.json、.py、.kt、.xml、.csv、.log、.yaml、.toml、.html、.css、.js 等）：以等宽字体显示可滚动的文本内容
- **图片文件**（.png、.jpg、.jpeg、.gif、.webp、.bmp）：以可缩放的图片形式展示
- **其他文件**：显示文件元数据（名称、大小、类型、路径、最后修改时间），提供"分享"操作，但不支持内联预览

#### 3. 文件操作

每个文件可用的操作：
- **分享**：通过 Android 分享菜单将文件发送到其他应用（在列表条目和预览界面均可使用）
- **删除**：通过确认对话框删除文件
- **复制路径**：将文件的内部路径复制到剪贴板（便于在提示词中引用）

目录可用的操作：
- **删除**：删除目录及其所有内容（确认对话框会显示包含的条目数量）

### 导航入口

- **设置/工具区域**：应用设置或主导航中的"文件"菜单项
- **聊天集成**：当 AI 保存文件时，聊天消息中包含可点击的文件引用，直接打开文件预览

### 用户交互流程

```
1. 用户从导航菜单打开文件浏览器
2. 文件浏览器显示根目录（user_files/）内容
3. 用户点击某个目录进入
4. 用户点击某个文件打开预览
5. 用户可在预览界面进行分享、删除或复制路径操作
6. 用户通过面包屑或返回按钮进行导航
```

## 验收标准

### TEST-025-01: 打开文件浏览器
- **Given** 用户在应用中
- **When** 他们导航到文件浏览器
- **Then** 文件浏览器打开，显示根目录内容

### TEST-025-02: 显示文件和目录
- **Given** user_files 目录中存在文件和目录
- **When** 文件浏览器加载完成
- **Then** 目录排在前面并带有文件夹图标，文件排在后面并带有类型图标，每个条目显示名称、大小和最后修改日期

### TEST-025-03: 进入目录
- **Given** 用户正在查看包含子目录的目录
- **When** 他们点击某个子目录
- **Then** 浏览器进入该目录，面包屑随之更新

### TEST-025-04: 面包屑导航
- **Given** 用户已导航进入嵌套目录（例如：根目录 > scripts > python）
- **When** 他们点击面包屑中的"scripts"
- **Then** 浏览器返回到"scripts"目录

### TEST-025-05: 预览文本文件
- **Given** 文件浏览器中存在一个 .txt 或 .py 文件
- **When** 用户点击该文件
- **Then** 预览界面打开，以等宽字体显示文件内容

### TEST-025-06: 预览图片文件
- **Given** 文件浏览器中存在一个 .png 或 .jpg 文件
- **When** 用户点击该文件
- **Then** 预览界面打开，以支持缩放和平移的方式显示图片

### TEST-025-07: 分享文件
- **Given** 用户正在查看文件预览
- **When** 他们点击分享操作
- **Then** Android 分享菜单打开，附带该文件

### TEST-025-08: 删除文件
- **Given** 用户正在浏览器中查看某个文件
- **When** 他们点击删除并确认
- **Then** 文件从存储中移除，并从列表中消失

### TEST-025-09: 删除目录
- **Given** 用户选择了一个目录进行删除
- **When** 他们确认删除对话框（对话框显示包含的条目数量）
- **Then** 该目录及其所有内容被删除

### TEST-025-10: 空状态
- **Given** user_files 目录中没有文件
- **When** 文件浏览器打开
- **Then** 显示空状态提示："暂无文件"

### TEST-025-11: 复制路径
- **Given** 用户正在查看文件预览
- **When** 他们点击"复制路径"
- **Then** 文件的内部路径被复制到剪贴板，并通过 toast 提示确认

### TEST-025-12: 聊天文件引用
- **Given** AI Agent 在聊天过程中使用 write_file 保存了一个文件
- **When** 保存成功
- **Then** 聊天消息中包含一个可点击的文件标签，点击后直接打开文件预览

## 非功能性需求

- 对于包含最多 100 个条目的目录，文件列表应在 200ms 内加载完成
- 对于不超过 1MB 的文本文件，文件预览应在 500ms 内加载完成
- 图片预览应支持双指缩放和平移手势
- 文件浏览器应能优雅处理文件被外部（如系统）删除的情况
- 支持预览的最大文件大小：文本文件 1MB，图片文件 10MB

## 范围外

- 浏览应用私有存储以外的文件（设备文件系统）
- 在文件浏览器中创建或编辑文件（文件仅由 AI 创建）
- 文件搜索或筛选
- 文件重命名或移动操作
- 多文件选择批量操作
- 文件的云备份或同步
- 文件加密
