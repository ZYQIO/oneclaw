## 用户故事

**作为** OneClawShadow 的用户，
**我希望** AI 智能体能够利用其提供商内置的搜索能力在网络上进行搜索，
**以便** 我能够获得基于实时最新信息的回答，并附有可验证的来源引用。

### 典型场景

1. 用户询问智能体"Android 16 的最新动态是什么？"——智能体使用提供商内置的网络搜索查找最新文章，并在回答中附上引用来源。
2. 用户为以研究为主的智能体配置文件启用网络搜索，使该智能体的所有对话自动获得实时网络数据访问能力。
3. 用户在智能体的回答中看到内联引用，点击来源链接即可在浏览器中打开原始文章。
4. 用户为专注于编程的智能体关闭网络搜索，以节省费用并避免不必要的搜索查询。

## 功能描述

### 概述

提供商网络搜索利用各 AI 模型提供商提供的内置搜索能力——包括 OpenAI 的网络搜索、Anthropic 的服务端网络搜索工具，以及 Google Gemini 的 Google 搜索接地功能。与应用现有的客户端工具 `webfetch` 和 `browser`（由 AI 显式调用以抓取特定 URL）不同，提供商网络搜索是一种服务端能力，由提供商的基础设施执行网络搜索，并自动将相关结果注入模型的上下文中。

与客户端工具相比，这种方式具有以下优势：
- **更低延迟**：搜索在提供商的基础设施上运行，避免额外的网络往返
- **更好的集成**：提供商控制搜索结果注入模型上下文的方式
- **自动引用**：提供商返回结构化引用数据，与回答的特定部分对应
- **无工具往返**：对于 OpenAI 和 Gemini，搜索在单次 API 调用中完成，无需工具调用循环

### 详细规格

#### 1. 单智能体网络搜索开关

每个智能体在其配置中拥有一个"网络搜索"开关：
- **默认**：关闭（禁用网络搜索）
- **开启时**：适配器在 API 请求中包含提供商特定的搜索配置
- 开关状态以布尔字段存储在 Agent 模型上

#### 2. 提供商特定行为

| 提供商 | 机制 | 搜索触发方式 | 费用 |
|--------|------|-------------|------|
| OpenAI | Chat Completions 中的 `web_search_options` | 启用后始终搜索 | $0.03-0.05/次搜索 |
| Anthropic | `web_search_20250305` 服务端工具 | 模型自主决定是否搜索 | $0.01/次搜索 |
| Gemini | tools 中的 `google_search` | 模型自主决定是否搜索 | $0.014/次查询 |

- **OpenAI**：需要支持搜索的模型（例如 `gpt-4o-search-preview`）。启用网络搜索后，模型始终会执行搜索。
- **Anthropic**：网络搜索工具会被添加到 tools 数组中。Claude 根据查询内容自主决定是否进行搜索。单次对话轮次中可能发生多次搜索。
- **Gemini**：Google 搜索接地功能会被添加到 tools 数组中。Gemini 自主决定是否使用搜索结果对回答进行接地。

#### 3. 引用展示

当 AI 回答中包含来自网络搜索的引用时：
- 回答文本下方显示"来源"区域
- 每个来源显示：标题（可点击）和域名
- 点击来源会在设备默认浏览器中打开对应 URL
- 小链接图标标注来源数量

#### 4. 搜索指示器

当提供商正在执行网络搜索时：
- 聊天界面中显示"正在搜索网络..."指示器（类似思考指示器）
- 这主要适用于 Anthropic，其搜索过程以服务端工具调用的形式可见

### 导航入口

- **智能体编辑页面**：智能体配置表单中的网络搜索开关
- **聊天页面**：AI 回答消息中内联展示的引用

### 用户交互流程

```
1. 用户打开智能体设置并启用"网络搜索"开关
2. 用户与智能体开始对话
3. 用户发送一条受益于网络信息的消息
4. 系统向提供商发送启用搜索的请求
5. 提供商执行网络搜索（如适用）
6. AI 回答以包含搜索接地内容的形式流式返回
7. 引用显示在回答文本下方
8. 用户点击引用以打开来源 URL
```

## 验收标准

### TEST-031-01：为智能体启用网络搜索
- **前提** 用户正在编辑某个智能体的配置
- **当** 用户将"网络搜索"切换为开启并保存
- **则** 智能体的 webSearchEnabled 设置已被持久化

### TEST-031-02：OpenAI 搜索请求
- **前提** 使用 OpenAI 提供商且已启用网络搜索的智能体
- **当** 用户发送一条消息
- **则** API 请求包含 `web_search_options` 字段

### TEST-031-03：Anthropic 搜索请求
- **前提** 使用 Anthropic 提供商且已启用网络搜索的智能体
- **当** 用户发送一条消息
- **则** API 请求的 tools 数组中包含 `web_search_20250305` 工具

### TEST-031-04：Gemini 搜索请求
- **前提** 使用 Gemini 提供商且已启用网络搜索的智能体
- **当** 用户发送一条消息
- **则** API 请求的 tools 数组中包含 `google_search`

### TEST-031-05：展示引用
- **前提** 提供商返回带有引用数据的回答
- **当** 回答在聊天界面中渲染
- **则** 回答文本下方出现"来源"区域，显示来源标题和域名

### TEST-031-06：点击引用链接
- **前提** 引用显示在某条回答下方
- **当** 用户点击某个引用
- **则** 该来源 URL 在设备默认浏览器中打开

### TEST-031-07：Anthropic 的搜索指示器
- **前提** 已启用网络搜索的 Anthropic 智能体
- **当** 模型发起服务端网络搜索
- **则** 搜索过程中出现"正在搜索网络..."指示器

### TEST-031-08：网络搜索默认关闭
- **前提** 新创建的智能体
- **当** 用户查看该智能体的配置
- **则** 网络搜索开关默认为关闭状态

### TEST-031-09：禁用时不进行搜索
- **前提** 已禁用网络搜索的智能体
- **当** 用户发送一条消息
- **则** API 请求不包含任何搜索配置

### TEST-031-10：引用跨会话持久化
- **前提** 包含引用的回答已被保存
- **当** 用户重新打开该会话
- **则** 引用仍正确显示在回答下方

## 非功能性需求

### 性能
- 网络搜索给整体响应时间带来的额外延迟不应超过 3 秒（取决于提供商）
- 引用渲染不应影响聊天滚动性能

### 费用意识
- 智能体配置中应注明网络搜索会产生额外 API 费用
- 费用提示应在开关附近可见

### 兼容性
- 网络搜索依赖于具体提供商；三个提供商均应显示该开关，但需优雅处理不支持的模型

## 范围外

- 域名过滤或允许/屏蔽域名列表（未来增强）
- 用于本地化搜索结果的用户位置配置（未来增强）
- 回答文本中的内联脚注式引用标记（v1 仅展示来源区域）
- 搜索查询可见性（展示提供商搜索了什么内容）
- AI 综合回答之前的搜索结果预览/摘要
- 提供商搜索不可用时的客户端网络搜索回退

## 错误处理

### 错误场景

1. **提供商不支持所选模型的网络搜索**
   - 请求在不含搜索的情况下继续执行；不向用户显示错误
   - 开关保持可用状态，但搜索功能不会激活

2. **搜索配额超限（速率限制）**
   - 提供商返回不含搜索结果的部分回答
   - 按原样展示回答，不显示引用

3. **搜索暂时不可用**
   - 对于 Anthropic：响应中返回 `web_search_tool_result_error`
   - 正常展示 AI 的回答；显示搜索不可用的简短提示
