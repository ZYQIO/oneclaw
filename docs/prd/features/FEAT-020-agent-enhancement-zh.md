## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 能够更好地控制 Agent 管理——包括可正常使用的模型选择器、对所有自定义 Agent 提供 Clone/Delete 功能，以及通过提示词创建 Agent 的能力，
**以便** 我能高效地创建、定制和管理 Agent，以满足我的需求。

### 典型场景
1. 用户打开某个自定义 Agent 的详情页，希望 Clone 它以创建一个变体——但目前没有 Clone 按钮。
2. 用户尝试为 Agent 选择首选模型，但选择器对交互没有响应。
3. 用户希望通过用自然语言描述 Agent 的用途来快速创建 Agent，而不是手动填写所有字段。

## 功能描述

### 概述
本功能针对现有 Agent 管理（FEAT-002）的三个改进方向：

1. **修复首选模型选择器** -- 当前的模型选择 UI 显示了可用模型，但不支持正常选择。替换为功能完整的下拉选择器。
2. **为所有 Agent 提供 Clone 和 Delete** -- Clone 按钮应在所有 Agent（内置和自定义）上可用。Delete 按钮应在所有自定义 Agent 上可用。自定义 Agent 同时显示 Clone 和 Delete。
3. **基于提示词的 Agent 创建** -- 新增一种创建流程，用户用自然语言描述想要的 Agent 类型，系统使用 AI 生成 Agent 的名称、描述和系统提示词。用户可在保存前预览和编辑。

### 详细规格

#### 1. 首选模型选择器修复

当前状态：`PreferredModelSelector` 组合函数将模型名称显示为 `TextButton` 条目，但交互不直观，在某些上下文中可能无法正常工作。

目标状态：替换为 `ExposedDropdownMenuBox`（Material 3 下拉菜单），功能如下：
- 将当前选中项或"使用全局默认"作为触发显示
- 打开一个按 Provider 分组的可滚动下拉列表，列出所有可用模型
- 允许清除选择，恢复为全局默认
- 对内置 Agent 禁用（只读）

#### 2. Clone 和 Delete 按钮逻辑

当前按钮可见性：

| Agent 类型 | Clone | Delete |
|-----------|-------|--------|
| 内置 | 是 | 否 |
| 自定义（已保存） | 否 | 是 |
| 自定义（新建/未保存） | 否 | 否 |

目标按钮可见性：

| Agent 类型 | Clone | Delete |
|-----------|-------|--------|
| 内置 | 是 | 否 |
| 自定义（已保存） | 是 | 是 |
| 自定义（新建/未保存） | 否 | 否 |

关键规则：
- 内置 Agent：仅 Clone（不可 Delete）
- 自定义 Agent（已保存）：Clone 和 Delete 均有
- 新建 Agent（尚未保存）：两者均无（请先保存）

#### 3. 基于提示词的 Agent 创建

在现有手动创建基础上新增的创建流程：

```
1. 用户导航至 Agent 列表页
2. 用户点击"创建 Agent"按钮
3. 系统显示创建 Agent 页面（与现在相同），但顶部增加
   "从提示词生成"区域
4. 用户输入对所需 Agent 的描述（例如："一个帮助调试和代码审查的
   Python 编程助手"）
5. 用户点击"生成"
6. 系统将描述发送给已配置的 AI 模型，并生成：
   - Agent 名称
   - Agent 描述
   - Agent 系统提示词
7. 生成的字段填充到下方表单中
8. 用户审阅并可选择编辑生成的内容
9. 用户点击"保存"以创建 Agent
```

生成所用的提示词模板在应用中硬编码。它指示 AI 生成一个包含 `name`、`description` 和 `systemPrompt` 字段的 JSON 对象。

## 验收标准

必须通过（全部必填）：
- [ ] 首选模型选择器使用正确的下拉菜单，能响应用户点击
- [ ] 首选模型下拉列表显示所有已启用 Provider 的可用模型
- [ ] 首选模型可清除并恢复为"使用全局默认"
- [ ] 首选模型选择器对内置 Agent 禁用
- [ ] Clone 按钮在自定义（已保存）Agent 上可见
- [ ] Clone 按钮在内置 Agent 上可见
- [ ] Delete 按钮仅在自定义（已保存）Agent 上可见
- [ ] 内置 Agent 不显示 Delete 按钮
- [ ] 新建（未保存）Agent 既不显示 Clone 也不显示 Delete
- [ ] Clone 自定义 Agent 后，新 Agent 名称为"Copy of [原名称]"
- [ ] "从提示词生成"输入框在创建 Agent 页面上可见
- [ ] 用户可输入提示词并点击"生成"以自动填充 Agent 字段
- [ ] 生成的名称、描述和系统提示词填充到表单中
- [ ] 用户可在保存前编辑生成的内容
- [ ] 生成过程中显示加载指示器，等待 AI 响应
- [ ] 生成失败时显示错误消息，且不丢失用户已输入的内容

可选（锦上添花）：
- [ ] "从提示词生成"功能也可在现有 Agent 编辑页面用于重新生成系统提示词
- [ ] 提示词历史记录 / 常见 Agent 类型建议

## UI/UX 要求

### 首选模型选择器
- 使用 Material 3 `ExposedDropdownMenuBox` 模式
- 触发器显示当前模型，格式为"Provider / 模型名称"或"使用全局默认"
- 下拉条目按 Provider 分组，以 Provider 名称作为分组标题
- 设置最大下拉高度，支持模型数量较多时滚动
- 下拉列表顶部提供"清除"选项，以重置为全局默认

### Clone 和 Delete 按钮
- 两个按钮均出现在 Agent 详情表单底部的操作区域
- Clone 按钮：中性/默认颜色，文本为"Clone Agent"
- Delete 按钮：错误/红色，文本为"Delete Agent"
- 当两个按钮均显示时，Clone 在 Delete 上方
- Delete 确认对话框与现有实现相同

### 从提示词生成区域
- 位于创建 Agent 页面顶部，在名称字段上方
- `OutlinedTextField`，标签为"描述您想要创建的 Agent..."
- 多行，显示 3-5 行
- "生成"按钮位于文本框右侧或下方
- 加载状态：按钮显示圆形进度指示器，文本框变为只读
- 生成完成后：下方表单字段填充内容，提示词字段保留可见并保留原始文本

## 功能边界

### 包含范围
- 修复首选模型下拉交互
- 为所有已保存 Agent 提供 Clone 按钮
- 仅为自定义已保存 Agent 提供 Delete 按钮
- 在创建 Agent 页面提供基于提示词的 Agent 创建功能
- 从用户描述 AI 生成名称、描述和系统提示词

### 不包含范围
- 基于提示词编辑现有 Agent（未来增强）
- 多轮对话式 Agent 精调
- Agent 模板库或市场
- 根据提示词自动选择工具（工具仍为手动选择，暂缓）

## 业务规则

### 模型选择器规则
1. 下拉列表中仅显示已启用（激活）Provider 的模型
2. 如果当前选中模型的 Provider 变为未激活，选择持续保留但显示警告
3. 清除首选模型意味着 Agent 使用全局默认模型

### Clone 规则
1. Clone 任意 Agent（内置或自定义）都会创建一个新的自定义 Agent
2. Clone 名称为"Copy of [原名称]"
3. Clone 继承：描述、系统提示词、首选模型/Provider
4. Clone 不继承：isBuiltIn 标志（始终为 false）
5. Clone 完成后，用户导航回 Agent 列表（现有行为）

### 提示词生成规则
1. 生成需要至少一个已激活且配置了 API Key 的 Provider
2. 用于生成的模型为全局默认模型
3. 如果生成失败，表单字段不做修改
4. 生成的系统提示词应根据用户描述具体、详尽
5. 用户的提示词描述不作为 Agent 的一部分保存

## 错误处理

### 错误场景

1. **尝试从提示词生成时未配置任何活跃 Provider**
   - 显示："未配置 AI Provider，请先设置一个 Provider。"
   - 用户操作：导航至 Provider 管理

2. **AI 生成失败（网络错误、API 错误）**
   - 显示："Agent 生成失败，请重试或手动填写字段。"
   - 用户操作：重试或手动填写表单

3. **AI 返回格式错误的响应**
   - 显示：与上述错误相同
   - 处理方式：解析可解析的部分，其余字段留空

4. **Clone 失败（存储错误）**
   - 显示："Agent Clone 失败，请重试。"
   - 用户操作：重试

## 测试要点

### 功能测试
- 验证首选模型下拉菜单可打开并列出所有活跃 Provider 的模型
- 验证从下拉菜单选择模型后，Agent 的首选模型更新
- 验证清除首选模型功能正常
- 验证内置 Agent 的下拉菜单被禁用
- 验证 Clone 按钮在已保存自定义 Agent 上可见
- 验证 Clone 按钮在内置 Agent 上可见
- 验证 Delete 按钮在内置 Agent 上不可见
- 验证 Delete 按钮在新建未保存 Agent 上不可见
- 验证基于提示词的生成可填充名称、描述和系统提示词
- 验证生成加载状态
- 验证生成错误处理

### 边界情况
- 使用极短输入从提示词生成（例如："助手"）
- 使用极长输入从提示词生成
- 无活跃 Provider 时尝试生成
- Clone 一个本身就是 Clone 的 Agent
- 选择某模型后，停用其 Provider，再打开 Agent 详情
