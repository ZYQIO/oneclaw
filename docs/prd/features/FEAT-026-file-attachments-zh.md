## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 能够在聊天消息中附加图片、视频和文件，
**以便** 向 AI 模型发送视觉和文档上下文以供分析，并在对话中以内联方式查看附加的媒体内容。

### 典型使用场景

1. 用户用相机拍摄一份文档照片，并将其发送给 AI 进行 OCR 或摘要提取。
2. 用户从相册中选择一张图片，并请 AI 对其进行描述或分析。
3. 用户从相册中附加一段视频，并请 AI 进行分析（Gemini），或收到当前提供商不支持视频的提示通知。
4. 用户附加一个 PDF 或文本文件，并请 AI 对其内容进行摘要。
5. 用户点击消息气泡中的图片，以全屏方式查看并支持缩放和平移。
6. 用户点击消息气泡中的视频缩略图，通过系统视频播放器进行播放。

## 功能描述

### 概述

FEAT-026 为聊天界面新增多模态输入支持。用户可通过聊天输入区的附件按钮，向消息中附加图片、视频和文件。附加的媒体内容在发送前以预览形式展示，发送后以内联方式渲染在消息气泡中。用户可点击媒体内容，以全屏方式查看图片，或通过系统播放器播放视频。

本功能同时扩展了 API 适配器层，以支持三个提供商（OpenAI、Anthropic、Gemini）的多模态内容，并在提供商不支持某种媒体类型时进行优雅降级处理。

### 详细说明

#### 附件按钮与选择器

- 聊天输入操作行新增附件按钮（"+"），位于技能按钮的右侧
- 点击按钮后，弹出底部表单，提供四个选项：
  - **照片**：从设备相册选择图片（Photo Picker API）
  - **视频**：从设备相册选择视频（Photo Picker API）
  - **相机**：使用设备相机拍摄照片（不支持录制视频）
  - **文件**：从设备中选择任意文件（SAF 文档选择器）
- 可同时从相册中选择多张图片或多个视频
- 选中的文件将被复制到应用内部存储路径 `files/attachments/{sessionId}/`

#### 附件预览（发送前）

- 选择文件后，文本输入框上方会出现一个可横向滚动的预览行
- 每个预览项显示：
  - **图片**：缩略图，带移除（X）按钮
  - **视频**：缩略图，带时长浮层和移除按钮
  - **文件**：文件名、大小和移除按钮
- 用户可在发送前单独移除某个附件

#### 消息气泡展示（发送后）

- 用户消息气泡中将以内联方式展示附加媒体：
  - **图片**：渲染为圆角缩略图（最大宽度约 240dp），显示在文本内容上方
  - **视频**：缩略图，带播放按钮浮层
  - **文件**：卡片式展示，含文件图标、文件名和大小
- 多个附件在气泡内以垂直列表方式排列

#### 媒体查看器（点击交互）

- **图片**：打开全屏图片查看器，支持双指缩放和平移手势
- **视频**：通过 Intent 打开系统视频播放器
- **文件**：通过 Intent 打开对应的系统应用（ACTION_VIEW）

#### 提供商兼容性

发送含附件的消息前，系统会检查当前提供商是否支持每种附件类型：

| 附件类型 | OpenAI | Anthropic | Gemini |
|---------|--------|-----------|--------|
| 图片（JPEG、PNG、GIF、WebP） | Yes | Yes | Yes |
| 视频 | No | No | Yes |
| 文件（PDF 等） | No（仅文本提取） | Yes（PDF via base64） | Yes |

若当前提供商不支持某种附件类型，系统将通过 Snackbar 提示用户该附件将被跳过。消息仍会携带文本和所有受支持的附件正常发送。

### 用户交互流程

#### 附加并发送图片

```
1. 用户点击聊天输入区的"+"按钮
2. 底部表单弹出，显示 照片 / 视频 / 相机 / 文件 选项
3. 用户点击"照片"
4. 系统 Photo Picker 打开
5. 用户选择一张或多张图片
6. 图片被复制到内部存储
7. 预览缩略图出现在文本输入框上方
8. 用户输入消息（可选）并点击发送
9. 消息携带文本和图片附件发送至 AI 提供商
10. 消息气泡在文本上方以内联方式展示图片
11. AI 回复到达（可能引用图片内容）
```

#### 查看附加图片

```
1. 用户点击消息气泡中的图片
2. 全屏图片查看器打开
3. 用户可双指缩放和平移
4. 用户点击返回或向下滑动以关闭
```

#### 不支持的附件类型警告

```
1. 用户当前选择的是 OpenAI 提供商
2. 用户附加了一段视频
3. 用户点击发送
4. Snackbar 提示："该提供商不支持视频附件，视频将被跳过"
5. 消息仅携带文本内容发送（视频被排除）
6. 视频附件仍保存在本地，并在消息气泡中显示
```

## 业务规则

### 功能规则

1. 附件始终复制到应用内部存储；初次复制后，应用不再引用外部 URI
2. 每个附件使用 UUID 文件名以避免冲突：`{uuid}.{extension}`
3. 图片和视频的缩略图在复制时生成，并与原文件存储在同一位置
4. 一条消息可包含 0 个或多个附件；V1 阶段对附件数量不设硬性上限
5. 当提供商不支持某种附件类型时，该附件被跳过并显示 Snackbar 警告，消息仍携带受支持的附件和文本正常发送
6. 若所有附件均不受支持但有文本内容，消息以纯文本形式发送
7. 若所有附件均不受支持且无文本内容，发送操作被阻止并显示错误提示
8. 附件文件按会话存储：`files/attachments/{sessionId}/`
9. 删除会话时，级联删除 `files/attachments/{sessionId}/` 目录

### 数据规则

- 单个文件最大大小：20MB（在选择时强制校验）
- 支持的图片 MIME 类型：`image/jpeg`、`image/png`、`image/gif`、`image/webp`
- 支持的视频 MIME 类型：`video/mp4`、`video/webm`、`video/quicktime`
- 文件类型：任意 MIME 类型（通用文件处理）
- 缩略图：JPEG 格式，最大 256x256 像素

### 权限规则

- 相机：需要 `android.permission.CAMERA`（运行时请求）
- 照片/视频选择器：无需权限（Android 13+ 使用 Photo Picker API，更低版本使用 READ_MEDIA_IMAGES/READ_MEDIA_VIDEO）
- 文件选择器：无需权限（SAF 自行处理访问授权）

## 非功能性需求

### 性能

- 文件复制到内部存储：20MB 以内文件 < 2 秒
- 缩略图生成：每张图片/视频 < 500ms
- base64 编码：20MB 以内文件 < 1 秒
- 附件预览渲染：< 100ms（使用预生成缩略图）
- 含附件的消息气泡：60fps 流畅滚动（使用 Coil 进行懒加载）

### 内存

- 缩略图：256x256 JPEG 约 ~256KB 每张
- 内存中 base64 编码文件：约为原始文件大小的 ~1.3 倍（临时占用，仅在 API 调用期间）
- 图片查看器：同一时刻仅加载单张全分辨率图片

### 兼容性

- Photo Picker：Android 13+（原生），11-12（通过 ActivityResultContracts 向后兼容）
- 相机拍照：所有支持的 Android 版本
- SAF 文档选择器：所有支持的 Android 版本

## 错误处理

### 错误场景

1. **文件过大**
   - 原因：选择的文件超过 20MB 限制
   - 处理：显示 Snackbar"文件过大（最大 20MB）"，不将其添加至附件列表

2. **文件复制失败**
   - 原因：存储空间不足或 I/O 错误
   - 处理：显示 Snackbar"附件保存失败"，不将其添加至附件列表

3. **相机不可用**
   - 原因：设备无相机或相机权限被拒绝
   - 处理：显示 Snackbar"相机不可用"，或发起权限请求

4. **缩略图生成失败**
   - 原因：图片或视频文件损坏
   - 处理：使用通用占位图标；文件仍可作为附件添加

5. **base64 编码失败**
   - 原因：文件在选择后到发送前被删除
   - 处理：跳过该附件，提示用户，发送剩余内容

6. **提供商 API 拒绝**
   - 原因：文件超出提供商限制或格式不受支持
   - 处理：在 AI 回复中报告错误（提供商错误信息）

## 数据需求

### 附件数据模型

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| id | String (UUID) | Yes | 附件唯一标识符 |
| messageId | String | Yes | 外键，关联 messages 表 |
| type | Enum | Yes | IMAGE、VIDEO、FILE |
| fileName | String | Yes | 原始文件名 |
| mimeType | String | Yes | MIME 类型（如 image/jpeg） |
| fileSize | Long | Yes | 文件大小（字节） |
| filePath | String | Yes | 文件在内部存储中的路径 |
| thumbnailPath | String | No | 缩略图路径（图片/视频） |
| width | Int | No | 图片/视频宽度（像素） |
| height | Int | No | 图片/视频高度（像素） |
| durationMs | Long | No | 视频时长（毫秒） |
| createdAt | Long | Yes | 时间戳 |

### 数据存储

- **元数据**：Room 数据库 `attachments` 表，外键关联 `messages` 表
- **文件**：应用内部存储路径 `files/attachments/{sessionId}/{uuid}.{ext}`
- **缩略图**：应用内部存储路径 `files/attachments/{sessionId}/thumbs/{uuid}_thumb.jpg`
- **保留策略**：父会话被删除时，对应文件同步删除
