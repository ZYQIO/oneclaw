## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 在一个地方查看所有可用工具并全局启用或禁用它们，
**以便** 我可以控制哪些工具对所有 Agent 可用，而无需逐个配置每个 Agent。

### 典型场景

1. 用户想了解 App 提供了哪些工具。他们打开设置，点击"管理工具"，看到所有工具的完整列表——包括内置 Kotlin 工具、JavaScript 扩展工具、工具组以及 `load_skill` 工具——并显示各工具的名称、描述和启用/禁用状态。
2. 用户决定不希望 AI 发起 HTTP 请求。他们打开工具列表，找到 `http_request`，将其关闭。此后，无论每个 Agent 如何配置，任何 Agent 都无法使用 `http_request`。
3. 用户在启用某个工具前想先了解它的功能。他们在列表中点击工具名称，进入详情视图，查看工具的描述、参数 Schema、所需 Android 权限、超时时间、来源类型及所属工具组（如有）。
4. 用户重新启用之前禁用的工具。该工具立即对所有在每个 Agent 配置中选中了它的 Agent 恢复可用。
5. 开发者向工具目录中添加了若干自定义 JS 工具。他们打开工具管理界面，确认所有工具均已正确加载，并查看其参数 Schema。
6. 用户希望一次性禁用一整组相关工具。他们找到该工具组（例如"Google Drive"），关闭组开关。组内所有工具立即被禁用，无需逐一操作。
7. 用户希望在保留组内其他工具的同时只禁用特定工具。他们展开该组，然后按需关闭个别工具的开关。

## 功能描述

### 概述

工具管理提供一个集中式界面，用于查看和控制 App 中注册的所有工具。目前，工具仅能通过每个 Agent 的配置（FEAT-002）进行查看，且无法全局禁用某个工具或查看其完整详情（参数、权限、超时时间）。本功能在设置界面新增"管理工具"入口，点击后进入专用工具管理界面。

核心设计原则是**全局启用/禁用**：全局禁用的工具对所有 Agent 完全不可用，与 Agent 各自的工具选择无关。当工具处于全局启用状态时，它对各 Agent 的可用性仍由 FEAT-002 中现有的每个 Agent 工具配置决定。

### 工具启用/禁用模型

系统为每个工具引入全局启用/禁用标志：

```
全局状态：已启用   + 每个 Agent：已选中   = 工具对该 Agent 可用
全局状态：已启用   + 每个 Agent：未选中   = 工具对该 Agent 不可用
全局状态：已禁用   + 每个 Agent：已选中   = 工具对该 Agent 不可用
全局状态：已禁用   + 每个 Agent：未选中   = 工具对该 Agent 不可用
```

换言之：
- 全局禁用是绝对覆盖——禁用的工具对工具执行系统不可见。
- 全局启用是前提条件——每个 Agent 的配置提供第二层控制。

### 启用/禁用状态的存储

每个工具的全局启用/禁用状态存储在本地键值存储（SharedPreferences 或类似机制）中，以工具名称为键。所有工具的默认状态为**已启用**——新注册的工具（包括重新加载后新添加的 JS 工具）默认为启用状态。这确保了向后兼容性：在用户明确禁用某工具之前，现有行为保持不变。

对于工具组（RFC-018），还需存储组级别的启用/禁用状态。键格式为 SharedPreferences 中的 `tool_group_enabled_{group_name}`（boolean 类型）。单个工具的键仍为 `tool_enabled_{tool_name}`。新工具组默认为**已启用**。

### 工具列表界面

工具管理界面显示当前在 `ToolRegistry` 中注册的所有工具。工具按三级层次结构布局：

#### 1. 内置工具区域

随 App 内置的单文件工具（来自 FEAT-015 的 Kotlin 或内置 JS 工具）。平铺显示（不可折叠）：

| 工具 | 描述 |
|------|-------------|
| `get_current_time` | 获取当前日期和时间 |
| `http_request` | 向 URL 发起 HTTP 请求 |
| `load_skill` | 加载技能的完整提示词指令 |
| `read_file` | 读取文件内容 |
| `webfetch` | 获取网页并返回内容 |
| `write_file` | 向文件写入内容 |

每个工具显示：名称、单行描述、"Built-in" 来源标签及单独的启用/禁用开关。

#### 2. 工具组区域

从数组清单文件（RFC-018）加载的工具。每个工具组是一个**可折叠区域**，具有以下属性：

| 属性 | 描述 |
|----------|-------------|
| 组名称 | 从清单文件名派生（例如 `google_drive.json` -> "Google Drive"） |
| 工具数量标签 | 显示组内工具数量（例如"3 tools"） |
| 组级别开关 | 一次性启用/禁用组内所有工具 |
| 折叠/展开控件 | 箭头图标，用于展开/折叠；**默认折叠** |

展开后，组内每个工具显示：
- 工具名称、单行描述、"Tool Group" 来源标签及单独的启用/禁用开关。

#### 3. 独立工具区域

不属于任何工具组的单文件 JS 扩展工具。平铺显示（不可折叠），布局与内置工具区域相同，但来源标签为 "JS Extension"。

#### 组开关与单个工具开关的关系

组开关与单个工具开关遵循父子关系，类似于 Android 权限组的逻辑：

- **组关闭**：所有子工具被禁用。组内的单个工具开关以灰色显示，无法交互。
- **组开启**：每个子工具可独立切换。
- **所有子工具均关闭**：当用户手动禁用组内所有单个工具时，组开关自动关闭。
- **组重新启用**：当组开关重新打开时，单个工具恢复到之前保存的状态（不会全部强制开启）。

每个区域内的工具按名称字母顺序排列。

### 工具详情视图

点击列表中的工具后，进入详情视图，展示工具的完整信息：

| 字段 | 数据来源 | 描述 |
|-------|--------|-------------|
| 名称 | `ToolDefinition.name` | 工具唯一标识符 |
| 描述 | `ToolDefinition.description` | 人类可读的描述 |
| 参数 | `ToolDefinition.parametersSchema` | 列出每个参数的名称、类型、描述、必填/可选标志、枚举值（如有）及默认值（如有） |
| 所需权限 | `ToolDefinition.requiredPermissions` | 工具所需的 Android 权限列表（例如 `READ_EXTERNAL_STORAGE`），或显示"None" |
| 超时时间 | `ToolDefinition.timeoutSeconds` | 最大执行时间（秒） |
| 来源 | 从工具加载来源派生 | "Built-in"（从应用 assets 加载或基于 Kotlin 实现）、"JS Extension"（从设备文件系统加载的单文件工具）或 "Tool Group"（从数组清单文件加载） |
| 所属组 | 组名称或 "None" | 工具所属的工具组（如从数组清单加载），独立/内置工具显示 "None" |
| 已启用 | 全局状态 | 当前全局启用/禁用状态及切换开关 |

对于 JS Extension 和 Tool Group 工具，还会额外显示：
- **文件路径**：设备上 `.js` 文件（或工具组的清单 `.json` 文件）的路径

### 与每个 Agent 工具选择（FEAT-002）的集成

Agent 配置（FEAT-002）中现有的每个 Agent 工具选择功能保持不变，仅做一处修改：

- 在 Agent 详情/编辑界面的工具列表中，全局禁用的工具以视觉方式标注（例如显示为灰色并附有"Globally disabled"标签），其复选框处于禁用状态，用户无法为 Agent 选择全局禁用的工具。
- 当 `ToolExecutionEngine` 为会话解析工具集时，它会过滤掉所有全局禁用的工具，即使 Agent 配置中包含这些工具。

### 与工具执行的集成

`ToolExecutionEngine` 已对请求的工具进行存在性校验并确认其对当前 Agent 可用。本功能在此基础上新增一项检查：

```
1. 模型请求工具调用（工具名称 + 参数）
2. 引擎检查工具是否存在于注册表中                    [现有]
3. 引擎检查工具是否已全局启用                        [新增]
4. 引擎检查工具是否在当前 Agent 的工具集中            [现有]
5. 引擎验证参数                                      [现有]
6. 引擎检查权限                                      [现有]
7. 引擎执行工具                                      [现有]
```

如果工具已全局禁用而模型请求调用它，引擎返回错误："Tool [name] is globally disabled and not available."

### 用户交互流程

#### 查看工具列表
```
1. 用户打开设置
2. 用户点击"Manage Tools"
3. 系统显示工具管理界面，列出所有已注册工具
4. 工具按三个区域组织：内置工具（平铺）、工具组（可折叠）、独立工具（平铺）
5. 每个工具显示名称、描述、来源标签及启用/禁用开关
6. 每个工具组显示组名称、工具数量标签、组开关及折叠/展开控件
```

#### 全局禁用工具
```
1. 用户打开工具管理界面
2. 用户找到要禁用的工具（例如 http_request）
3. 用户点击切换开关将其禁用
4. 开关切换为关闭状态
5. 短暂显示 Snackbar："http_request disabled"
6. 该工具立即对所有 Agent 不可用
7. 如果某 Agent 已将此工具选中，它仍保留在配置中，但处于非激活状态
```

#### 禁用工具组
```
1. 用户打开工具管理界面
2. 用户找到要禁用的工具组（例如"Google Drive"）
3. 用户点击组级别的切换开关将其禁用
4. 组开关切换为关闭状态
5. 组内所有单个工具开关变为灰色且不可操作
6. 短暂显示 Snackbar："Google Drive group disabled"
7. 组内所有工具立即对所有 Agent 不可用
```

#### 展开工具组并切换单个工具
```
1. 用户打开工具管理界面
2. 用户点击工具组的展开控件（箭头）
3. 工具组展开，显示组内所有单个工具
4. 用户点击组内某个工具的开关将其禁用
5. 该工具开关切换为关闭状态
6. 短暂显示 Snackbar："[tool_name] disabled"
7. 如果用户已禁用组内所有工具，组开关自动关闭
```

#### 重新启用工具组
```
1. 用户打开工具管理界面
2. 用户找到已禁用的工具组（组开关为关闭状态）
3. 用户点击组开关将其启用
4. 组开关切换为开启状态
5. 组内单个工具开关恢复为可交互状态
6. 每个工具恢复到之前保存的单独状态（不会全部强制开启）
7. 短暂显示 Snackbar："Google Drive group enabled"
```

#### 查看工具详情
```
1. 用户打开工具管理界面
2. 用户点击工具名称（非开关部分）
3. 系统显示工具详情视图
4. 用户可查看：名称、描述、所有参数及其类型、权限、超时时间、来源及所属工具组
5. 用户也可在此视图中切换启用状态
6. 用户点击返回，回到工具列表
```

#### 重新启用已禁用的工具
```
1. 用户打开工具管理界面
2. 用户找到已禁用的工具（开关显示为关闭状态）
3. 用户点击切换开关将其启用
4. 开关切换为开启状态
5. 短暂显示 Snackbar："http_request enabled"
6. 该工具立即对在每个 Agent 配置中已选中它的 Agent 恢复可用
```

## 业务规则

### 启用/禁用规则
1. 所有工具在首次注册时默认为启用状态。
2. 全局启用/禁用状态以工具名称为键分别存储。
3. 禁用工具立即生效——此后对该工具的任何调用都将失败并返回错误，即使在正在进行的对话中亦然。
4. 启用工具立即生效——在每个 Agent 配置中已选中该工具的 Agent 可在下次工具调用时使用它。
5. 如果某工具被注销（例如 JS 工具文件被删除）后以相同名称重新注册，之前存储的启用/禁用状态将被保留。
6. 启用/禁用状态与工具来源（内置、JS 扩展、工具组或 Skill 相关）无关。

### 工具组开关规则
1. 新工具组在首次加载时默认为启用状态。
2. 组级别启用/禁用状态以 `tool_group_enabled_{group_name}` 为键存储在 SharedPreferences 中。
3. 组开关关闭时，组内所有子工具变为禁用状态。单个工具开关变为灰色且不可交互。
4. 组开关开启时，每个子工具可独立切换。单个工具恢复到之前保存的状态（不会全部强制开启）。
5. 当用户手动禁用已启用组内的所有单个工具时，组开关自动关闭。
6. 当组从自动关闭（规则 5 触发）状态重新启用时，单个工具恢复到最后一次明确保存的状态。
7. 组名称从清单文件名派生：去掉扩展名，将下划线替换为空格，每个单词首字母大写（例如 `google_drive.json` -> "Google Drive"）。
8. 禁用工具组立即生效——组内所有工具对所有 Agent 不可用。

### 与每个 Agent 配置的关系
1. 全局禁用无条件覆盖每个 Agent 的工具选择。
2. 全局启用不会自动将工具添加到任何 Agent 的工具集中——每个 Agent 的配置仍是精细控制层。
3. 当工具被全局禁用时，它仍显示在 Agent 配置工具列表中，但以视觉方式标注为已禁用，且复选框无法操作。
4. 重新启用工具不会改变任何 Agent 的每个 Agent 工具选择——Agent 之前的配置保持不变。

### 显示规则
1. 工具列表在界面打开时显示 ToolRegistry 中当前已注册的所有工具。
2. 如果工具被重新加载（例如 FEAT-012 的 JS 工具重载），工具列表在下次访问界面时更新。
3. 工具名称按注册时的原样显示（遵循 FEAT-004 的 snake_case 命名规范）。
4. 工具描述按 ToolDefinition 中的原样显示。
5. 来源类型从工具的加载来源派生：从应用 assets（`js/tools/`）加载或以 Kotlin 实现的工具（如 `LoadSkillTool`）为 "Built-in"；从数组清单文件（RFC-018）加载的工具为 "Tool Group"；从设备文件系统（`/sdcard/OneClawShadow/tools/` 或 `{app_files}/tools/`）加载的不属于任何工具组的单文件工具为 "JS Extension"（独立工具）。RFC-015 之后，除 `LoadSkillTool` 外的所有内置工具均为 JavaScript 实现，因此仅凭类类型无法区分内置工具和用户自定义 JS 工具。
6. 工具组按组名称字母顺序排列。每个组内的工具按工具名称字母顺序排列。
7. 如果没有工具组存在，"TOOL GROUPS" 区域标题不显示。
8. 如果没有独立 JS 扩展工具存在，"STANDALONE" 区域标题不显示。

## 非功能性要求

### 性能
- 工具管理界面加载时间 < 200ms（从内存中的 ToolRegistry + 本地 SharedPreferences 读取）
- 启用/禁用开关持久化时间 < 50ms（SharedPreferences 写入）
- 工具详情视图渲染时间 < 100ms
- 工具组展开/折叠动画完成时间 < 200ms
- 所有工具管理操作均无需网络请求

### 数据
- 单个工具启用/禁用状态存储在本地 SharedPreferences 中（键：`tool_enabled_{tool_name}`，值：boolean）
- 组启用/禁用状态存储在本地 SharedPreferences 中（键：`tool_group_enabled_{group_name}`，值：boolean）
- 工具启用/禁用配置视情况纳入数据同步/备份（FEAT-007）
- 无需新增 Room 实体

### 可靠性
- 如果启用/禁用存储损坏或不可用，所有工具默认为启用状态（故障开放）
- 如果组启用/禁用状态损坏或不可用，工具组默认为启用状态（故障开放）
- 工具管理界面优雅处理空 ToolRegistry（显示空状态提示信息）

### 无障碍
- 所有 UI 元素具备内容描述，支持屏幕阅读器
- 切换开关播报当前状态（"enabled"/"disabled"）
- 工具组展开/折叠状态播报（"expanded"/"collapsed"）
- 工具名称和描述在大字体下清晰可读
- 启用/禁用状态不仅依赖颜色表示（开关位置提供非颜色提示）
- 已禁用组内的灰色工具对屏幕阅读器播报为"disabled by group"

## 错误处理

### 错误场景

1. **SharedPreferences 写入失败**
   - 原因：存储空间不足或持久化启用/禁用状态时发生 I/O 错误
   - 处理：显示错误 Snackbar："Failed to save tool state. Please try again."，开关恢复至之前的状态

2. **ToolRegistry 为空**
   - 原因：无工具注册（例如启动失败）
   - 处理：在工具管理界面显示空状态："No tools available."

3. **工具在详情视图打开时被注销**
   - 原因：用户查看某工具详情时，JS 工具重载移除了该工具
   - 处理：显示提示信息"This tool is no longer available"，并返回工具列表

4. **模型调用全局禁用的工具**
   - 原因：AI 模型请求调用用户已禁用的工具
   - 处理：向模型返回错误："Tool [name] is globally disabled and not available."
   - 模型可向用户告知情况或尝试其他方案

5. **工具组清单文件在显示时被移除**
   - 原因：工具组的清单文件被删除或变为不可读
   - 处理：在下次访问界面或工具重载时，该组及其工具从列表中移除。如果被移除组中某工具的详情视图正在打开，显示"This tool is no longer available"并返回

### 边界情况

- 禁用所有工具：所有开关关闭，Agent 只能进行无工具调用的聊天，不崩溃。
- 快速重复切换开关：无状态损坏，以最后一次切换结果为准。
- 工具管理界面打开时触发 JS 工具重载：界面在下次导航或刷新时更新。
- 无参数的工具：详情视图显示"No parameters"而非空区域。
- 无所需权限的工具：详情视图权限字段显示"None"。
- 描述非常长的工具：列表中截断显示，详情视图中显示完整文本。
- 在工具管理界面旋转屏幕：状态、滚动位置和工具组展开/折叠状态保持不变。
- 工具数量非常多（50+）：列表滚动流畅，无性能下降。
- 工具名称包含特殊字符：正确显示。
- 仅包含一个工具的组：以可折叠组形式显示（与多工具组相同）。
- 在已有工具组的情况下添加新的组清单文件：新组在下次访问界面时显示在工具组区域。
- 组清单文件被移除：该组及其工具在下次访问界面或重载时消失。
- 空工具组（清单文件存在但不包含任何工具）：该组不显示在列表中。
