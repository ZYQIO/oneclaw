## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 能够安排 AI 代理任务在指定时间自动运行，
**以便** 我无需手动干预，就能自动化诸如晨间简报、每日摘要或定期数据检查等重复性任务。

### 典型场景

1. 用户创建一个每天早上 7:00 运行的晨间简报任务。每天早晨，代理自动创建一个新会话，通过工具获取新闻和天气，并向用户发送带有响应预览的通知。
2. 用户为下周二下午 3:00 设置一个一次性提醒任务，要求代理准备会议记录。任务触发一次后，创建包含结果的会话，然后自动禁用自身。
3. 用户创建一个每周任务，在每周一早上 9:00 运行以生成每周摘要。代理运行完整的代理循环（包括工具调用），将结果保存到会话中，并通知用户。
4. 用户通过切换开关临时禁用定时任务。闹钟被取消。当重新启用时，闹钟将为下一个触发时间重新注册。
5. 用户重启设备。所有已启用的定时任务自动重新注册闹钟，并在正确时间继续触发。

## 功能描述

### 概述

定时任务允许用户创建、管理和自动化基于时间计划的 AI 代理任务。每个任务与一个代理和一条提示消息相关联。当计划时间到达时，系统自动创建一个新会话，运行完整的代理循环（包括流式传输和工具执行），保存结果，并发送通知。

### 计划类型

1. **一次性**：在指定日期和时间触发一次。执行后自动禁用。
2. **每日**：每天在指定的小时和分钟触发。
3. **每周**：每周在指定的星期几和指定的小时分钟触发。

### 任务生命周期

1. **创建**：用户填写任务名称，选择代理，编写提示，选择计划类型和时间。
2. **调度**：系统计算下一次触发时间，并通过 AlarmManager 注册精确闹钟。
3. **触发**：AlarmManager 触发 BroadcastReceiver，后者将 WorkManager 一次性工作请求加入队列。
4. **执行**：Worker 创建会话，运行 SendMessageUseCase，收集完整响应，并更新执行状态。
5. **通知**：Worker 发送包含任务名称和响应预览的通知。点击通知将打开对应的会话。
6. **重新调度**：对于重复任务（DAILY、WEEKLY），Worker 计算并注册下一个闹钟。对于 ONE_TIME 任务，任务自动被禁用。

### 管理

- **列表视图**：显示所有定时任务，包含名称、计划描述、启用/禁用切换开关以及最后执行状态。
- **创建/编辑**：表单包含任务名称、代理选择器、提示文本、计划类型选择器、时间选择器，以及（对于 ONE_TIME）日期选择器或（对于 WEEKLY）星期几选择器。
- **删除**：取消闹钟并从数据库中移除任务。
- **切换**：启用/禁用开关，用于注册或取消闹钟。

## 验收标准

### TEST-019-01：创建一次性任务
- **前提条件** 用户位于定时任务创建界面
- **当** 用户填写名称、选择代理、输入提示、选择"一次性"、选取未来的日期和时间，并点击保存
- **则** 任务出现在列表中，显示正确的计划描述和已启用的切换开关

### TEST-019-02：任务触发并创建会话
- **前提条件** 定时任务已启用且触发时间已到
- **当** AlarmManager 触发闹钟
- **则** 创建一个新会话，代理循环使用配置的提示运行，响应被保存在会话中

### TEST-019-03：完成后发送通知
- **前提条件** 定时任务已完成执行
- **当** Worker 完成工作
- **则** 发送包含任务名称和响应预览的通知。点击通知将打开该会话。

### TEST-019-04：一次性任务自动禁用
- **前提条件** 一次性定时任务已执行
- **当** 执行完成
- **则** 任务自动禁用，不再安排后续闹钟

### TEST-019-05：每日重新调度
- **前提条件** 每日定时任务已在早上 7:00 执行
- **当** 执行完成
- **则** 下一个闹钟注册为次日早上 7:00

### TEST-019-06：切换启用/禁用
- **前提条件** 定时任务已启用
- **当** 用户将其关闭
- **则** 闹钟被取消。当重新打开时，闹钟将为下一个触发时间重新注册。

### TEST-019-07：重启后持久化
- **前提条件** 存在已启用的定时任务
- **当** 设备重启
- **则** 所有已启用的闹钟重新注册

### TEST-019-08：删除任务
- **前提条件** 定时任务存在
- **当** 用户将其删除
- **则** 闹钟被取消，任务从数据库中移除

## 非功能性需求

- 任务执行必须使用前台服务通知，以防止系统在长时间运行的代理循环期间终止 Worker。
- 闹钟使用 `setExactAndAllowWhileIdle` 以确保在 Doze 模式下也能可靠触发。
- 任务执行需要网络连接（WorkManager 约束条件）。
- 任务执行超时：最长 10 分钟。

## 范围外

- 自定义 cron 表达式（仅支持预设计划类型）
- 任务链或任务间依赖关系
- 在执行过程中编辑正在运行任务的参数
- 基于云端的调度（所有调度均通过 AlarmManager 在本地完成）
