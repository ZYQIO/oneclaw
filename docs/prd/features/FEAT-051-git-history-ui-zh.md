# FEAT-051: Git 历史记录浏览器 UI

## 文档信息
- **功能 ID**: FEAT-051
- **状态**: 草稿
- **创建日期**: 2026-03-02
- **最后更新**: 2026-03-02

## 背景

RFC-050 为 `filesDir` 下的所有文本文件引入了自动 git 版本控制。对记忆文件、每日日志和 AI 生成的 Markdown 的每次写入都会自动提交。然而，目前应用内没有供用户浏览此历史记录的 UI。用户必须让 AI 运行 `git_log` 或 `git_show` 工具来查看历史版本，这种方式既间接又不易被发现。

## 目标

1. 添加一个 Git 历史记录界面，按时间倒序列出所有提交
2. 允许用户按子目录筛选提交（例如，仅显示记忆变更、仅显示每日日志）
3. 点击某条提交可查看其完整 diff，让用户清楚了解具体发生了哪些变更
4. 可通过记忆界面上的专属入口访问

## 非目标

- UI 中不提供还原/回滚操作（已可通过 AI 的 `git_restore` 工具实现）
- 不提供分支管理 UI
- 不提供 diff 语法高亮（纯等宽字体文本已足够）
- 不提供超出简单"加载更多"机制的分页功能

## 用户故事

**US-1: 浏览完整历史**
作为用户，我希望能从记忆界面打开"版本历史"视图，并看到对我文件所做的所有变更的时间顺序列表。

**US-2: 按类别筛选**
作为用户，我希望能筛选历史记录，仅显示记忆变更、仅显示每日日志或仅显示 AI 生成的文件，以便专注于我所关心的内容。

**US-3: 查看具体变更**
作为用户，我希望能点击某条提交条目，查看添加或删除了哪些文本的可读 diff，从而了解具体发生了什么变化。

## 功能需求

### FR-1: 入口
在记忆界面顶部栏添加"版本历史"图标按钮。点击后导航至 Git 历史记录界面。

### FR-2: 提交列表
- 按时间倒序显示提交（最新的排在最前）
- 每条记录显示：
  - 短 SHA（7 个字符）
  - 相对或绝对时间戳（例如"2 小时前"或"2026-03-02 14:30"）
  - 提交信息
- 默认限制：50 条提交；点击"加载更多"按钮可追加显示后续 50 条

### FR-3: 路径筛选
在列表顶部提供分段控件或标签行，包含以下筛选选项：

| 标签 | 传入 `git_log` 的路径筛选 |
|---|---|
| 全部 | 无 |
| 记忆 | `memory/MEMORY.md` |
| 每日日志 | `memory/daily/` |
| 文件 | 无，但排除带 `memory/` 前缀的提交 |

### FR-4: 提交详情
点击某条提交后，打开详情面板（底部弹出面板或全屏），显示：
- 提交 SHA、日期、提交信息
- 在可滚动的等宽字体 `Text` composable 中渲染完整 diff 文本
- "关闭"按钮

### FR-5: 空状态
如果仓库尚无任何提交，显示占位信息："No version history yet."

## 非功能需求

- 所有 `AppGitRepository` 调用须通过 ViewModel 在 IO dispatcher 上执行
- 加载提交时界面必须保持响应（显示加载指示器）
- diff 视图必须能优雅处理较大的 diff（支持滚动，对合理大小（最大约 50KB）的内容不截断）
