## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 拥有一个现代、视觉精致的聊天输入区域，风格与 Google Gemini 应用保持一致，
**以便** 撰写和发送消息的体验更加高端、直观，并与当代移动端 AI 聊天界面的设计风格相符。

### 典型场景
1. 用户打开对话，看到一个圆角胶囊形输入框，背景为柔和的填充色，而非锐利的描边边框，整体感觉现代且友好。
2. 用户开始输入消息。输入框从单行起步，随着文字换行而平滑向上扩展，最多约 6 行，让用户有足够的空间撰写较长的消息，而不会产生突兀的布局跳动。
3. 用户点击底部操作行左侧的技能按钮，打开技能选择面板，选择一个技能后，斜杠命令被插入输入框——所有现有的技能与斜杠命令功能均与之前保持一致。
4. 用户正在聊天，同时 AI 正在流式输出回复。停止按钮出现在底部操作区右侧，用户点击后停止生成，该按钮随即恢复为发送按钮。
5. 用户以"/"作为第一个字符开始输入，斜杠命令弹窗出现在输入区域上方，行为与现在完全一致，无任何变化。

## 功能描述

### 概述
本功能是对 `ChatScreen.kt` 中 `ChatInput` Composable 的视觉重新设计。当前输入区域使用 `OutlinedTextField` 包裹在一个具有色调高程（tonal elevation）的 `Surface` 中，操作按钮与文本输入框排列在同一水平行内。本次重新设计将其替换为 Google Gemini 风格的输入区域：一个填充式、大圆角的容器，文本输入框位于上方，操作按钮行位于下方。这是一次纯粹的视觉与布局变更——不涉及任何数据层、ViewModel 或架构上的改动。

### 当前实现

当前 `ChatInput` Composable 的结构如下：
- 一个 `tonalElevation = 2.dp` 的 `Surface`，内部包含一个 `Row`
- `Row` 内从左至右依次为：技能按钮、`OutlinedTextField`、停止按钮（条件显示）、发送按钮
- `OutlinedTextField` 使用 `MaterialTheme.shapes.extraLarge` 并设置 `maxLines = 6`
- 所有元素纵向对齐方式为 `Alignment.Bottom`

### 新设计：Gemini 风格输入框

重新设计后的输入区域在圆角容器内采用上下两层的竖向布局：

#### 第一层：文本输入区域
- 以 `BasicTextField`（或无边框装饰的 `TextField`）替代 `OutlinedTextField`
- 背景为填充式 Surface 颜色（如 `surfaceContainerHigh` 或 `surfaceContainer`），而非描边边框
- 整个容器采用较大的圆角半径（28dp），形成胶囊/药丸形状
- 占位提示文字（"Message or /skill"）在输入框内左对齐显示
- 输入框从 1 行开始，随用户输入自动向上扩展，最多约 6 行
- 当内容超过最大高度时，输入框变为可滚动

#### 第二层：底部操作行
- 位于文本输入框下方，处于同一圆角容器内
- 左侧：技能按钮（现有的 `AutoAwesome` 图标），并为未来的附件按钮预留空间
- 右侧：发送按钮（填充圆形，使用主色），或停止按钮（流式输出期间显示）
- 操作行的垂直内边距比文本输入区域更小，保持视觉上的紧凑感
- 文本输入区域与操作行之间通过适当的间距进行视觉分隔（不是硬性分割线）

#### 外部容器
- 外部容器替换当前具有 `tonalElevation` 的 `Surface`
- 背景：`MaterialTheme.colorScheme.surfaceContainerHigh`（填充式、略微凸显的 Surface，无描边轮廓）
- 形状：`RoundedCornerShape(28.dp)`——大圆角，在输入内容较少时呈现胶囊/药丸形状
- 水平外边距：距屏幕边缘 12dp
- 底部内边距：与当前保持一致，遵守导航栏 insets 和 IME padding

### 视觉对比

#### 之前（当前）

```
+------------------------------------------------------------------+
| Surface (tonalElevation = 2dp, full width)                       |
|                                                                  |
|  [Skill]  +---[OutlinedTextField]---+  [Stop?] [Send]            |
|    btn    | Message or /skill       |   btn     btn              |
|           +-------------------------+                            |
|                                                                  |
+------------------------------------------------------------------+
```

主要特征：
- 单行布局：按钮与文本输入框并排排列
- `OutlinedTextField`，带有可见的描边边框
- 全宽 Surface，矩形，带色调高程
- 按钮被挤压在文本输入框两侧

#### 之后（Gemini 风格）

```
   +------------------------------------------------------------+
   |  (surfaceContainerHigh, roundedCorner 28dp)                 |
   |                                                             |
   |   Message or /skill                                         |
   |   (auto-expanding text area, 1-6 lines)                     |
   |                                                             |
   |  -------  (subtle spacing / visual separator)  -------      |
   |                                                             |
   |   [Skill]  [Attach*]              [Stop?] [Send]            |
   |    btn      future                 btn     btn              |
   |                                                             |
   +------------------------------------------------------------+

   * Attach button is a placeholder for future functionality
```

主要特征：
- 上下两层竖向布局：文本区域在上，操作行在下
- 填充式背景，无描边
- 大圆角（胶囊形状）
- 距屏幕边缘有水平外边距（非全宽）
- 操作按钮有更多呼吸空间
- 视觉上更柔和、更现代

### 行为要求

所有现有行为必须完整保留：
1. **文本输入**：输入、清除、文字选择功能均与之前一致
2. **发送操作**：发送按钮触发 `onSend`，当文本为空或未配置 Provider 时禁用
3. **停止操作**：流式输出期间显示停止按钮，触发 `onStop`
4. **技能按钮**：触发 `onSkillClick` 以打开技能选择底部面板
5. **斜杠命令**：以"/"作为第一个字符输入时，在输入区域上方触发 `SlashCommandPopup`
6. **FocusRequester**：文本输入框仍接受 `FocusRequester` 以支持编程式聚焦（如选择技能后）
7. **IME padding**：输入区域随软键盘弹出进行适配（`imePadding()`）
8. **导航栏 padding**：输入区域遵守导航栏 insets（`navigationBarsPadding()`）
9. **自动扩展**：文本输入框从 1 行扩展至最多 6 行，超过后变为可滚动

### Composable 签名

`ChatInput` Composable 的签名保持不变：

```kotlin
@Composable
fun ChatInput(
    text: String,
    onTextChange: (String) -> Unit,
    onSend: () -> Unit,
    onStop: () -> Unit,
    onSkillClick: () -> Unit,
    isStreaming: Boolean,
    hasConfiguredProvider: Boolean,
    focusRequester: FocusRequester
)
```

无需新增参数，无需修改 ViewModel，无需变更状态。这仅是 Composable 内部的布局变更。

### 主题一致性

重新设计后的输入框必须使用应用现有的 Material 3 主题：
- **主色**（金色/琥珀色 `#6D5E0F`）：用于发送按钮的容器颜色
- **surfaceContainerHigh**：用于输入区域背景
- **onSurface**：用于输入文字颜色
- **onSurfaceVariant**：用于占位文字及图标着色
- **errorContainer**：用于停止按钮容器（不变）
- 所有颜色均来自 `MaterialTheme.colorScheme`，不使用硬编码值

## UI/UX 要求

### 布局规格

#### 外部容器
- 背景：`MaterialTheme.colorScheme.surfaceContainerHigh`
- 形状：`RoundedCornerShape(28.dp)`
- 水平外边距：距每侧屏幕边缘 12dp
- 底部外边距：导航栏上方 8dp（在 `navigationBarsPadding()` 之后）
- 内部内边距：水平 12dp，垂直 8dp

#### 文本输入框
- 无边框，无描边
- 背景：透明（继承容器背景）
- 文字样式：`MaterialTheme.typography.bodyLarge`
- 文字颜色：`MaterialTheme.colorScheme.onSurface`
- 占位文字："Message or /skill"
- 占位文字颜色：`MaterialTheme.colorScheme.onSurfaceVariant`
- 最小高度：1 行（约 24dp 内容 + 内边距）
- 最大可见行数：6 行（超过后可滚动）
- 顶部内边距：12dp
- 底部内边距：4dp（位于操作行上方）

#### 操作行
- 高度：40dp（图标按钮触摸目标）
- 顶部内边距：4dp
- 底部内边距：8dp
- 左侧：技能按钮（24dp 图标，40dp 触摸目标）
- 左侧（未来）：附件按钮占位（视觉预留空间或隐藏）
- 右侧：发送按钮（40dp，填充圆形，主色容器）和/或停止按钮
- 左右之间：弹性间隔

#### 发送按钮
- 容器：填充圆形，`MaterialTheme.colorScheme.primary`
- 图标：`Icons.Default.Send`，着色：`MaterialTheme.colorScheme.onPrimary`
- 尺寸：40dp
- 禁用状态：降低透明度，或使用 `MaterialTheme.colorScheme.surfaceVariant` 容器

#### 停止按钮
- 容器：填充圆形，`MaterialTheme.colorScheme.errorContainer`
- 指示器：`CircularProgressIndicator`（18dp，2dp 描边，错误颜色）
- 尺寸：40dp
- 流式输出期间显示在发送按钮左侧，或替代发送按钮

#### 技能按钮
- 图标：`Icons.Default.AutoAwesome`
- 着色：`MaterialTheme.colorScheme.onSurfaceVariant`
- 尺寸：40dp 触摸目标，24dp 图标

## 功能边界

### 包含范围
- `ChatInput` Composable 布局与样式的视觉重新设计
- 将 `OutlinedTextField` 替换为填充式容器内的无边框文本输入框
- 从单行布局切换为上下两层（文本区域 + 操作行）的竖向布局
- 为容器应用大圆角（胶囊/药丸形状）
- 为容器添加水平外边距
- 调整按钮位置（技能按钮在左下，发送/停止按钮在右下）
- 更新聊天屏幕的 Roborazzi 截图基准

### 不包含范围（V1）
- 附件按钮功能（仅预留占位空间）
- 语音输入按钮
- 对 `ChatViewModel` 或聊天状态管理的任何改动
- 对 `SlashCommandPopup` 组件的任何改动
- 对 `SkillSelectionBottomSheet` 组件的任何改动
- 对消息气泡或消息列表的任何改动
- 对数据层、Repository 或数据库的任何改动
- 对顶部应用栏或导航抽屉的任何改动
- 超出 Material 3 主题自动提供范围的深色/浅色模式专项定制
- 自定义输入框装饰（如格式工具栏、表情符号选择器）

## 业务规则

### 布局规则
1. 文本输入框必须始终位于操作行上方（竖向堆叠）
2. 技能按钮始终位于操作行左侧
3. 发送按钮始终位于操作行右侧
4. 停止按钮仅在 `isStreaming` 为 true 时出现
5. 容器必须具有一致的圆角，不受内容高度影响
6. 容器不得为全宽，必须保留水平外边距（最少 12dp）

### 行为规则
1. 所有现有的 ChatInput 参数与回调必须以完全相同的方式工作
2. 该 Composable 不得引入新的状态或副作用
3. 键盘 inset 处理（imePadding、navigationBarsPadding）必须保留
4. 输入"/"时，SlashCommandPopup 必须继续出现在输入区域上方
5. FocusRequester 必须仍然绑定到文本输入框，以支持编程式聚焦

### 视觉规则
1. 所有颜色必须来自 `MaterialTheme.colorScheme`——不得使用硬编码颜色值
2. 所有字体排版必须来自 `MaterialTheme.typography`——不得使用硬编码文字样式
3. 所有形状必须使用 `RoundedCornerShape` 并指定对应半径，而非 `MaterialTheme.shapes`
4. 设计在浅色与深色主题下均须呈现正确效果（Material 3 通过色彩方案自动处理）

## 非功能性要求

### 性能
- 文本输入框对按键的响应不得有可感知的延迟（与当前保持一致）
- 高度自动扩展不得造成可见的卡顿或闪烁
- 输入过程中的布局重组应尽量减少（避免操作行发生不必要的重组）

### 兼容性
- 最低 API 级别：与应用当前最低要求保持一致（API 26）
- 必须在所有支持的屏幕尺寸上正确工作
- 必须同时支持横屏与竖屏方向
- 必须支持系统字体大小缩放（无障碍功能）
- 必须支持从右至左（RTL）布局

### 视觉质量
- 圆角容器在所有屏幕密度下渲染时不得出现锯齿伪影
- 发送按钮在启用/禁用状态之间的切换必须流畅
- 文本输入框扩展必须在视觉上平滑（如已实现动画，不得出现突然跳变）

### 边界情况
- 超长的单词且不含空格（文本输入框应通过换行或滚动处理水平溢出）
- 快速交替输入与发送（布局不出现异常）
- 输入文字时旋转设备（布局自适应，文字得以保留）
- 系统字体大小设置为最大（布局能容纳更大文字而不裁切）
- RTL 语言输入（布局正确镜像）
- 斜杠命令弹窗打开的同时键盘可见（弹窗保持在输入框上方、消息列表下方）
- 屏幕宽度极窄（容器外边距不应导致文本输入框变得无法使用）
- 快速切换流式输出与非流式输出状态（停止/发送按钮正确切换）
