## 用户故事

**作为** OneClawShadow 的用户，
**我想要** AI Agent 能调用工具来执行操作（读取文件、发起 HTTP 请求、查看时间等），
**以便** AI 不仅能聊天，还能代我执行实际操作。

### 典型场景
1. 用户问"现在几点了？"——AI 调用 `get_current_time` 工具，回复当前时间。
2. 用户说"读一下我的笔记文件"——AI 调用 `read_file` 并传入文件路径，工具从本地存储读取文件，AI 总结内容。
3. 用户说"查一下最新的比特币价格"——AI 调用 `http_request` 查询公开 API 并返回结果。
4. 用户要求 AI 执行一个需要链式调用多个工具的多步骤任务。
5. 某个工具需要 Android 存储权限——系统权限对话框弹出，用户授权后工具继续执行。

## 功能描述

### 概述
工具系统是一个可扩展的框架，允许 AI 模型执行文本生成以外的操作。它定义了工具注册、调用和结果回传的标准接口。系统处理完整的工具调用循环：模型请求工具调用 -> 工具执行 -> 结果返回给模型 -> 模型继续。架构上的优先级是清晰、定义良好、可扩展的接口——使得随时间添加新工具变得容易。V1 内置一小组入门工具。

### 架构原则

1. **接口优先**：工具注册和调用协议是最重要的设计决策。必须清晰、有类型、有版本。
2. **Provider 无关的工具定义**：工具使用统一格式定义一次（基于 JSON Schema，对齐 OpenAI function calling 格式）。Provider 适配层处理不同 API provider（OpenAI、Anthropic、Gemini）的格式转换。
3. **可扩展性优于完整性**：宁可工具少但接口出色，也不要工具多但系统僵化。
4. **默认自动执行**：工具无需用户确认即可运行，除非需要 Android 系统权限。

### 工具定义格式

每个工具使用以下结构定义，基于 JSON Schema（对齐 OpenAI function calling 格式）：

```json
{
  "name": "read_file",
  "description": "Read the contents of a file from local storage",
  "parameters": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "The absolute file path to read"
      }
    },
    "required": ["path"]
  }
}
```

这是统一的内部表示格式。向不同 provider 发送工具定义时，provider 适配层转换为 provider 的特定格式：
- **OpenAI**：直接使用（这是原生格式）
- **Anthropic**：轻微结构调整以匹配 Anthropic 的 tool use 格式
- **Gemini**：转换为 Gemini 的 function declaration 格式
- **自定义**：取决于用户选择的协议兼容类型

### 工具注册接口

每个工具必须实现：

| 组件 | 说明 |
|------|------|
| 名称 | 唯一标识符（例如 `read_file`）。遵循 snake_case 命名约定。 |
| 描述 | 人类可读的工具功能描述（一句话）。用于 AI 提示词中，并在 Agent 配置中展示给用户。 |
| 参数 Schema | 定义工具接受的输入参数的 JSON Schema。 |
| 所需权限 | 该工具需要的 Android 权限列表（如有）。例如 `ACCESS_FINE_LOCATION`。 |
| 超时 | 最大执行时间（秒）。默认：30 秒。可按工具覆盖。 |
| 执行函数 | 接收参数并返回结果的实际实现。 |

### 工具执行引擎

执行引擎管理完整的工具调用生命周期：

#### 执行流程
```
1. 模型响应包含工具调用请求（工具名 + 参数）
2. 引擎验证工具名存在且对当前 Agent 可用
3. 引擎根据工具的 schema 验证参数
4. 引擎检查工具是否需要 Android 权限
   - 如果需要权限且未授予：触发 Android 权限请求对话框
   - 如果用户拒绝权限：返回权限错误给模型
5. 引擎在后台线程上执行工具（非主线程/UI 线程）
6. 引擎应用超时——如果执行超过工具的超时时间，中止并返回超时错误
7. 引擎捕获结果（成功或错误）
8. 引擎将结果作为工具结果消息返回给模型
9. 模型处理结果并继续（可能发起更多工具调用或产生最终响应）
```

#### 执行环境
- 工具在应用进程内的后台线程上执行（非主线程/UI 线程）
- 工具执行期间 UI 保持响应
- 如果工具崩溃（未处理异常），错误被捕获并作为工具错误结果返回给模型——应用不会崩溃

#### 超时处理
- 每个工具有可配置的超时（默认：30 秒）
- 按工具类型的推荐默认值：
  - 简单查询（获取时间等）：5 秒
  - 本地文件操作：10 秒
  - HTTP/网络请求：30 秒
- 达到超时时：
  - 执行被中止
  - 超时错误返回给模型
  - 模型可以决定如何处理（重试、通知用户等）

#### 多工具和链式调用
- 单次模型响应可能请求多个工具调用（并行或顺序，取决于模型）
- 引擎执行所有调用并将全部结果返回给模型
- 模型可能根据结果发起额外的工具调用（链式调用）
- 此循环持续直到模型产生没有工具调用的最终文字响应
- 工具调用轮次没有硬性限制，但 token/费用追踪（FEAT-006）提供可见性

### Android 权限处理

一些工具需要 Android 系统权限。工具系统透明地处理这一点：

1. 每个工具在注册时声明其所需权限
2. 执行工具前，引擎检查所需权限是否已授予
3. 如果未授予：
   - 向用户显示标准 Android 权限请求对话框
   - 如果用户授权：工具执行继续
   - 如果用户拒绝：返回权限拒绝错误给模型
4. 权限授予由 Android 系统记忆（标准行为）
5. 如果权限之前被"不再询问"拒绝，引擎返回错误，建议用户在系统设置中启用权限

### 内置工具（V1）

V1 内置以下入门工具：

#### 1. `get_current_time`
- **描述**：获取当前日期和时间
- **参数**：
  - `timezone`（string，可选）：时区标识符（例如 "America/New_York"）。默认使用设备时区。
  - `format`（string，可选）：输出格式（例如 "ISO8601"、"human_readable"）。默认 ISO8601。
- **所需权限**：无
- **超时**：5 秒
- **返回**：当前日期时间字符串

#### 2. `read_file`
- **描述**：从应用私有存储读取文件内容
- **参数**：
  - `path`（string，必填）：要读取的文件路径（限定在应用私有存储内）
  - `encoding`（string，可选）：文件编码。默认 "UTF-8"。
- **所需权限**：无（文件访问限定在应用私有存储）
- **超时**：10 秒
- **返回**：文件内容字符串。对于二进制文件，返回错误建议使用其他方式。

#### 3. `write_file`
- **描述**：将内容写入应用私有存储的文件
- **参数**：
  - `path`（string，必填）：要写入的文件路径（限定在应用私有存储内）
  - `content`（string，必填）：要写入的内容
  - `mode`（string，可选）：写入模式——"overwrite"（默认）或 "append"
- **所需权限**：无（文件访问限定在应用私有存储）
- **超时**：10 秒
- **返回**：确认信息，包含文件路径和写入字节数

#### 4. `http_request`
- **描述**：向 URL 发起 HTTP 请求
- **参数**：
  - `url`（string，必填）：请求的 URL
  - `method`（string，可选）：HTTP 方法——"GET"（默认）、"POST"、"PUT"、"DELETE"
  - `headers`（object，可选）：HTTP 头的键值对
  - `body`（string，可选）：请求体（用于 POST/PUT）
- **所需权限**：`INTERNET`（Android 默认授予）
- **超时**：30 秒
- **返回**：响应状态码、响应头和响应体（过大时截断）

### 工具结果格式

返回给模型的工具结果遵循一致的格式：

**成功：**
```json
{
  "status": "success",
  "result": "实际结果数据"
}
```

**错误：**
```json
{
  "status": "error",
  "error_type": "timeout | permission_denied | execution_error | validation_error",
  "message": "人类可读的错误描述"
}
```

此格式不论调用哪个工具都保持一致。Provider 适配层将其转换为每个 provider 的适当工具结果格式。

### Agent 配置中的工具可见性

配置 Agent 的工具集时（FEAT-002），用户看到：
- 每个可用工具的名称和描述
- 为 Agent 启用/禁用每个工具的复选框
- 展示示例：
  ```
  [x] get_current_time - 获取当前日期和时间
  [x] read_file - 从本地存储读取文件内容
  [x] write_file - 将内容写入本地存储的文件
  [ ] http_request - 向 URL 发起 HTTP 请求
  ```
- 需要特殊权限的工具显示权限图标/指示
- 此层级不显示参数详情（对于选择界面来说过于冗长）

## 业务规则

### 工具规则
1. 每个工具必须有唯一名称（注册时强制执行）
2. 工具名称遵循 snake_case 命名约定
3. 工具默认自动执行，无需用户确认
4. 唯一面向用户的中断是 Android 系统权限对话框
5. 工具失败时返回错误给模型——由模型决定如何处理
6. 引擎绝不静默丢弃工具调用；每次调用都会得到结果（成功或错误）

### 执行规则
1. 工具始终在后台线程执行
2. 如果工具超过其超时时间，执行被中止并返回超时错误
3. 如果工具抛出未处理异常，异常被捕获并返回执行错误
4. 单次响应中的多个工具调用都会被执行（如果模型请求，可能并行执行）
5. 工具调用链深度没有硬性限制，但费用追踪提供可见性

### 权限规则
1. 工具在注册时声明所需权限
2. 每次执行前都检查权限（不仅是第一次）
3. 如果权限未授予，显示标准 Android 对话框
4. 如果被拒绝，返回清晰的错误给模型
5. 工具系统绝不绕过 Android 的权限模型

## 非功能性需求

### 性能
- 工具注册（应用启动时）对所有内置工具完成时间 < 100ms
- 工具执行开销（不含实际工作）< 10ms
- 工具结果序列化 < 10ms
- 工具执行期间 UI 保持响应（60fps）

### 可靠性
- 工具崩溃绝不导致应用崩溃
- 网络工具失败（超时、DNS 等）返回清晰错误
- 文件工具失败（权限、文件不存在等）返回清晰错误

### 安全
- 文件工具仅访问应用私有存储（context.filesDir），通过 FsBridge 允许列表验证路径
- HTTP 工具遵循 HTTPS；对用户指定的 HTTP 请求不做限制
- 工具结果可能包含敏感数据——它们与其他消息以相同的安全级别存储在会话历史中

## 错误处理

### 错误场景

1. **工具未找到**
   - 原因：模型请求了不存在或当前 Agent 不可用的工具
   - 处理：返回错误给模型："工具 [name] 不可用"
   - 模型可以通知用户或尝试其他方法

2. **参数验证失败**
   - 原因：模型提供了无效参数（类型错误、缺少必填字段）
   - 处理：返回带详情的验证错误给模型
   - 模型可以修正并重试

3. **权限拒绝**
   - 原因：用户拒绝了工具所需的 Android 权限
   - 处理：返回权限错误给模型："权限 [X] 被用户拒绝"
   - 模型可以通知用户

4. **执行超时**
   - 原因：工具超过其配置的超时时间
   - 处理：中止执行，返回超时错误给模型
   - 模型可以重试或通知用户

5. **工具崩溃（未处理异常）**
   - 原因：工具实现中的 bug
   - 处理：捕获异常，返回带错误信息的执行错误给模型
   - 应用继续正常运行

6. **文件未找到**
   - 原因：`read_file` 使用不存在的路径调用
   - 处理：返回错误给模型："文件未找到：[path]"

7. **HTTP 工具网络错误**
   - 原因：DNS 失败、连接超时、服务器错误
   - 处理：返回带 HTTP 状态码或网络错误详情的错误给模型

- 10+ 轮的工具调用链
