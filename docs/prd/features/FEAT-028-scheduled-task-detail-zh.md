## 用户故事

**作为** OneClawShadow 的用户，
**我希望** 查看定时任务的详细信息，包括其配置、执行历史记录和下次触发时间，
**以便** 我能够监控任务执行情况、诊断失败原因、回顾历史结果，并快速跳转到对应的执行会话，而无需进入编辑页面。

### 典型场景

1. 用户在列表中点击某个定时任务，一目了然地查看其完整配置（Agent 名称、提示词、计划详情）和当前状态，而无需进入编辑模式。
2. 用户想排查某个每日任务昨晚失败的原因。他们打开详情页，查看带有时间戳、状态标识和错误信息的执行历史，然后点击失败条目跳转到对应的会话，回顾对话内容。
3. 用户有一个每周报告任务，想在时区变更后确认下次触发时间是否正确。详情页显示精确的下次触发日期时间。
4. 用户想立即手动运行某个定时任务进行测试，而无需等到计划时间。他们在详情页点击"立即运行"。
5. 用户查看执行历史后发现某任务持续失败。从详情页点击"编辑"以调整提示词或切换到其他 Agent。

## 功能描述

### 概述

定时任务详情页是一个只读视图，展示单个定时任务的完整信息。它替代了当前在列表中点击任务直接跳转到编辑表单的行为。现在点击任务会打开此详情页，作为查看任务状态、浏览执行历史和执行快速操作的主要中心。

### 页面区域

#### 1. 任务配置区域

以可读、不可编辑的格式展示任务的静态配置：
- **任务名称**（页面标题）
- **Agent**：已解析的 Agent 名称（而非原始 ID）
- **提示词**：完整的提示词文本，内容较长时可展开
- **计划安排**：人类可读的描述（例如："每天 07:00"、"每周一 09:00"、"一次性，时间为 2026-03-15 14:30"）
- **启用/禁用**：当前状态，带有快速启用/禁用的切换开关

#### 2. 状态区域

展示运行时状态信息：
- **下次触发时间**：下次计划执行的格式化日期时间（对于已禁用或已完成的一次性任务，显示"已禁用"或"已完成"）
- **上次执行**：最近一次执行的时间和状态
- **创建时间 / 更新时间**：任务创建和最后修改的时间戳

#### 3. 执行历史区域

按时间倒序排列的所有历史执行记录，最新在前：
- **时间戳**：执行开始的时间
- **状态**：SUCCESS 或 FAILED 标识
- **会话链接**：可点击跳转到对应的聊天会话
- **时长**：执行所用时间（可选，如有记录）

此功能需要一个新的 `task_execution_records` 数据模型，因为当前的 `ScheduledTask` 只存储最后一次执行记录。历史记录将持久化到一个新的 Room 数据表中。

#### 4. 操作

详情页上可执行的操作：
- **编辑**：跳转到预填当前任务数据的编辑页面
- **删除**：删除任务（带确认对话框）
- **切换**：启用/禁用任务（内联开关）
- **立即运行**：手动立即触发任务执行，创建新会话并运行完整的 Agent 循环。适用于无需等待计划时间的测试场景。
- **查看上次会话**：快速跳转到最近一次执行的会话（如有）

### 导航流程变更

**变更前 (RFC-019)**：
```
列表 -> 点击条目 -> 编辑页面
```

**变更后 (FEAT-028)**：
```
列表 -> 点击条目 -> 详情页 -> 编辑按钮 -> 编辑页面
```

列表页面上的 FAB 仍直接跳转到创建页面。

## 验收标准

### TEST-028-01: 导航到详情页
- **Given** 用户在定时任务列表页面
- **When** 他们点击某个任务条目
- **Then** 详情页打开，显示该任务的完整配置、状态和执行历史

### TEST-028-02: 显示任务配置
- **Given** 用户在某任务的详情页
- **Then** 页面显示：任务名称、Agent 名称（非 ID）、完整提示词文本、计划描述和启用状态

### TEST-028-03: 显示下次触发时间
- **Given** 用户查看某个已启用的、计划在 08:00 执行的每日任务详情页
- **Then** 下次触发时间显示正确的即将到来的日期时间
- **And** 对于已禁用的任务，下次触发时间显示"已禁用"

### TEST-028-04: 显示执行历史
- **Given** 某定时任务已执行 3 次（2 次成功，1 次失败）
- **When** 用户打开详情页
- **Then** 3 条执行记录均显示，且时间戳、状态和会话链接均正确

### TEST-028-05: 跳转到执行会话
- **Given** 用户在详情页看到一条带有会话的执行历史条目
- **When** 他们点击该条目
- **Then** 应用跳转到对应的聊天会话，显示该次执行的对话内容

### TEST-028-06: 从详情页编辑
- **Given** 用户在详情页
- **When** 他们点击编辑操作
- **Then** 编辑页面打开，并预填任务当前配置

### TEST-028-07: 从详情页删除
- **Given** 用户在详情页
- **When** 他们点击删除并确认
- **Then** 任务被删除，闹钟被取消，用户返回列表页面

### TEST-028-08: 从详情页切换状态
- **Given** 用户在某个已启用任务的详情页
- **When** 他们将开关切换到禁用
- **Then** 闹钟被取消，状态更新为已禁用，下次触发时间相应更新

### TEST-028-09: 立即运行
- **Given** 用户在某任务的详情页
- **When** 他们点击"立即运行"
- **Then** 任务立即执行（创建会话、运行 Agent 循环），执行历史更新新条目，并发送结果通知

### TEST-028-10: 空执行历史
- **Given** 一个从未被执行过的新创建任务
- **When** 用户打开详情页
- **Then** 执行历史区域显示空状态提示，如"暂无执行记录"

## 非功能性需求

- 每个任务的执行历史最多保留最近 50 条，以防止无限增长。
- 超过 90 天的执行历史应自动清理。
- 对于最多拥有 50 条执行历史的任务，详情页应在 200ms 内完成加载。
- "立即运行"应显示进度指示器，并防止重复点击。

## 范围外

- 执行日志详情（在详情页展示完整对话记录——用户可跳转到会话查看）
- 执行历史的导出或分享
- 从历史记录中重试执行（重新运行某次历史执行）
- 按任务配置推送通知（全局通知设置由 FEAT-008 处理）
- 执行统计或图表（如成功率、平均时长）
