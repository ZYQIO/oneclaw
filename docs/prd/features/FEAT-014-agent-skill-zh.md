## 用户故事

**作为** OneClawShadow 的用户,
**我想要** 触发预定义的技能工作流,引导 AI 通过结构化的多步骤流程完成任务,
**以便** 我可以一致且高效地完成复杂或重复性任务,无需每次都编写详细的指令。

### 典型场景
1. 用户想要总结一个本地文件。不用写详细的提示词说明要读什么、怎么总结,只需输入 `/summarize-file` 并提供文件路径。AI 按照技能的结构化工作流读取文件并生成格式化的摘要。
2. 用户想检查设备存储状态。点击输入区域的 Skill 按钮,从列表中选择"存储检查",AI 按工作流收集并报告存储信息。
3. 用户正在与 AI 聊天时提到需要翻译一份文档。AI 识别到这匹配 `translate-file` 技能,主动通过 `load_skill` 工具加载技能提示词,并按照结构化翻译工作流执行。
4. 高级用户为自己的特定工作流创建自定义技能(如"日报生成器"),并导出为 Markdown 文件分享给同事。
5. 用户导入朋友分享的技能文件,该技能立即出现在技能列表中并可使用。

## 功能描述

### 功能概述
Agent Skill 引入了一个位于用户和工具系统之间的"提示词模板"层。Skill 是预定义的 Markdown 文件,包含结构化的指令,引导 AI 使用现有的工具按照特定工作流完成任务。Skill 本身不执行代码,而是通过向对话中注入详细的提示词指令来编排 AI 的行为。

该设计遵循 Claude Code 模式:系统提示词包含一个轻量级的可用技能注册表(名称 + 一句话描述),完整的技能内容在触发时按需加载。这样可以保持系统提示词紧凑,同时让技能可被发现。

### 核心概念: Skill = 提示词模板

Skill 定义为带有 YAML frontmatter 的 Markdown 文件:

```markdown
---
name: summarize-file
display_name: "总结文件"
description: "读取本地文件并生成结构化摘要"
version: "1.0"
tools_required:
  - read_file
parameters:
  - name: file_path
    type: string
    required: true
    description: "要总结的文件的绝对路径"
  - name: language
    type: string
    required: false
    description: "输出语言(默认:与源文件相同)"
---

# 总结文件

## 指令

1. 使用 `read_file` 工具读取 {{file_path}} 处的文件
2. 如果文件无法读取,清晰报告错误并停止
3. 分析内容并识别:
   - 文档的主题或目的
   - 关键要点和论据
   - 值得注意的数据、统计或结论
4. 生成结构化摘要:
   - **标题**: 推断的文档标题
   - **主题**: 1-2 句概述
   - **关键要点**: 要点列表(5-10 项)
   - **值得注意的细节**: 任何重要的具体内容
   - **摘要长度**: 保持在 500 字以内
5. 如果指定了 `language`,用该语言撰写摘要
```

### 系统提示词中的技能注册

系统提示词中包含所有可用技能的简要列表:

```
Available skills (use load_skill tool to load full instructions):
- summarize-file: Read a local file and produce a structured summary
- translate-file: Translate a file's content to a target language
- fetch-webpage: Fetch and summarize a webpage's content
- rewrite-text: Rewrite text in a specified style or tone
- device-info: Gather and report device information
```

此列表从技能 frontmatter 自动生成。每个技能一行,轻量化,让 AI 知道有哪些技能可用而不消耗过多 token。

### 技能加载机制: `load_skill` 工具

在工具系统中新增一个内置工具 `load_skill`:

| 组件 | 描述 |
|------|------|
| 名称 | `load_skill` |
| 描述 | 加载技能的完整提示词指令 |
| 参数 | `name` (字符串, 必填): 要加载的技能名称 |
| 所需权限 | 无 |
| 超时 | 5 秒 |
| 返回 | 技能的完整 Markdown 内容(frontmatter 之后的所有内容) |

`load_skill` 工具读取技能的 Markdown 文件,解析 frontmatter,在提供参数值时替换参数占位符,然后返回提示词内容。AI 随后遵循加载的指令执行。

### 三条触发路径

#### 路径 1: / 命令(输入框)
1. 用户在聊天输入框中输入 `/`
2. 弹出/自动完成列表显示可用技能(随用户输入实时过滤)
3. 用户选择技能(如 `/summarize-file`)
4. 如果技能有必填参数,显示简单的参数输入表单
5. 系统发送消息调用 `load_skill`,传入技能名称和参数
6. AI 接收完整技能提示词并按指令执行

#### 路径 2: UI 技能按钮
1. 用户点击技能按钮(位于输入框旁边或上方)
2. 底部弹出表或弹窗显示分类的技能列表
3. 用户选择技能
4. 如果技能有必填参数,显示参数输入表单
5. 从步骤 5 起与路径 1 相同

#### 路径 3: AI 自主调用
1. 对话过程中,AI 识别到用户的请求匹配某个可用技能
2. AI 调用 `load_skill` 传入技能名称
3. AI 遵循加载的指令执行
4. 用户在聊天中看到技能被加载(显示为工具调用)

### 技能与 Agent 的关系

技能**独立于 Agent**,是全局资源,任何 Agent 在任何会话中都可使用。这意味着:
- 无论当前活跃的是哪个 Agent,所有技能都可用
- 系统提示词中的技能注册表对所有 Agent 相同
- Agent 的工具集不影响技能的可用性(但如果技能需要的工具不在当前 Agent 的工具集中,执行可能失败 -- AI 应优雅地处理这种情况)

### 技能存储

每个技能存储为一个以技能名命名的目录中的 `SKILL.md` 文件(遵循 Claude Code 约定):

```
skills/
  summarize-file/
    SKILL.md
  translate-file/
    SKILL.md
  fetch-webpage/
    SKILL.md
  my-custom-skill/
    SKILL.md
```

目录名作为技能标识符。`SKILL.md` 文件包含完整的技能定义(YAML frontmatter + Markdown 提示词内容)。

#### 内置技能
- 作为 app 资产文件打包(在 `assets/skills/<skill-name>/SKILL.md` 中)
- 只读,用户不能修改或删除
- 随 app 更新而更新

#### 用户自定义技能
- 存储在 app 内部存储中(如 `files/skills/<skill-name>/SKILL.md`)
- 用户可以创建、编辑和删除
- 通过应用内 UI(技能编辑器)创建或从文件导入

#### 技能文件位置
- 内置: `assets/skills/summarize-file/SKILL.md`
- 用户自定义: `{app-internal}/skills/my-custom-skill/SKILL.md`

基于目录的结构也为未来扩展留有余地 -- 技能目录中可以在 `SKILL.md` 旁放置额外资源(如示例文件、多语言变体)。

### 用户自定义技能创建

用户可以通过 app 创建自定义技能:

1. 导航到技能管理界面
2. 点击"创建技能"
3. 填写:
   - 名称(必填,snake-case,用作标识符)
   - 显示名称(必填,人类可读)
   - 描述(必填,一句话)
   - 参数(可选,为每个参数定义名称/类型/是否必填/描述)
   - 提示词内容(必填,Markdown 格式的指令 -- 技能的核心)
   - 所需工具(可选,技能需要的工具列表)
4. 保存 -- 技能立即可用

高级用户也可以直接在技能目录中创建/编辑 `SKILL.md` 文件(通过文件系统或 `write_file` 工具)。

### 技能导入/导出/分享

#### 导出
- 用户可以将任何技能(内置或自定义)导出为其 `SKILL.md` 文件(未来可支持导出整个技能目录的 zip 包)
- 导出文件包含完整的技能定义(frontmatter + 内容)
- 用户可通过 Android 分享意图分享文件(邮件、即时通讯、文件传输等)

#### 导入
- 用户可以从任何来源导入 `SKILL.md` 文件
- App 验证文件结构(有效的 frontmatter、必填字段齐全)
- App 根据文件中的 `name` 字段创建对应目录,并将 `SKILL.md` 存入其中
- 如果同名技能已存在(即目录已存在),提示用户选择重命名或替换
- 导入的技能存储为用户自定义技能(可编辑)

### 技能在聊天中的显示

当技能被加载并执行时,聊天显示:
1. **技能加载指示**: 显示哪个技能被加载(如"已加载技能: 总结文件")
2. **工具调用**: 按 FEAT-001 中定义的常规工具调用显示
3. **技能结果**: AI 按照技能指令生成的回复(常规 AI 消息)

技能加载显示为常规工具调用(`load_skill`),保持与现有工具调用显示的 UI 一致性。

### 用户交互流程

#### 创建自定义技能
```
1. 用户导航到技能管理界面(通过设置或菜单)
2. 用户点击"创建技能"
3. 系统显示技能创建表单
4. 用户填写名称、显示名称、描述
5. 用户可选定义参数
6. 用户编写提示词内容(Markdown 编辑器)
7. 用户可选选择所需工具
8. 用户点击"保存"
9. 系统验证并保存技能
10. 技能立即在 / 命令和技能按钮中可用
```

#### 通过 / 命令使用技能
```
1. 用户在聊天输入框中输入"/"
2. 系统显示可用技能的自动完成列表
3. 用户选择或继续输入以过滤(如"/sum"显示"summarize-file")
4. 用户选择"summarize-file"
5. 系统显示必填参数的输入表单(如 file_path)
6. 用户提供文件路径
7. 系统发送消息:"Use the summarize-file skill on /path/to/file"
8. AI 调用 load_skill,接收指令,按工作流执行
9. AI 生成结构化摘要
```

#### 导入分享的技能
```
1. 用户收到 SKILL.md 文件(通过邮件、即时通讯等)
2. 用户用 OneClawShadow 打开文件(Android intent filter)
   或: 在技能管理界面使用"导入技能"选择文件
3. 系统解析并验证文件
4. 如果有效: 显示技能预览(名称、描述、内容)
5. 用户点击"导入"
6. 系统根据技能名称创建目录,将 SKILL.md 存入其中
7. 技能出现在技能列表中,立即可用
```

## UI/UX 要求

### 技能触发: / 命令
- 用户在输入框中输入 `/` 作为第一个字符时,在输入框上方显示弹出列表
- 列表显示技能显示名称和描述
- 随用户输入实时过滤(如 `/sum` 过滤出匹配"sum"的技能)
- 点击技能选择它
- 如果技能有必填参数,显示紧凑的内联表单或对话框
- `/` 触发仅在消息开头有效(避免干扰包含 `/` 的普通文本)

### 技能触发: UI 按钮
- 输入区域附近的小图标按钮(如闪电图标或类似图标)
- 点击打开底部弹出列表,列出所有可用技能
- 技能按类别分组(文件、网络、文本、系统)
- 内置技能与用户自定义技能有视觉区分
- 点击技能触发(如有必填参数则先提示输入)

### 技能管理界面
- 可从设置或专门的导航项进入
- 列表视图分两个部分:"内置"和"自定义"
- 每项显示:显示名称、描述、标记(内置/自定义)
- "创建技能"悬浮按钮或顶部操作按钮
- 点击查看/编辑,滑动或长按进行操作(删除、导出、克隆)

### 技能编辑器界面
- 创建/编辑技能的表单布局:
  - 名称(文本输入,强制 snake-case)
  - 显示名称(文本输入)
  - 描述(文本输入,单行)
  - 版本(文本输入,默认"1.0")
  - 参数部分(添加/删除参数条目,每个包含名称/类型/是否必填/描述)
  - 所需工具(从可用工具中多选)
  - 提示词内容(多行文本编辑器,支持 Markdown,大面积)
- 内置技能: 所有字段只读,带有"克隆"和"导出"按钮
- 自定义技能: 所有字段可编辑,带有"保存"、"删除"和"导出"按钮

### 交互反馈
- 技能加载: 聊天中显示工具调用
- 技能创建/保存: 成功提示条
- 技能删除: 确认对话框,然后成功提示条
- 技能导入: 成功提示条,显示技能名称
- 导入错误: 错误对话框,显示详情(无效格式、缺失字段等)

## 功能边界

### 包含的功能
- `load_skill` 内置工具,用于加载技能提示词内容
- 技能注册信息注入到系统提示词
- 聊天输入框中的 / 命令触发与自动完成
- UI 按钮触发与技能选择底部弹出表
- AI 通过 `load_skill` 工具自主调用
- 内置技能: 文件操作、网络/信息获取、文本处理、系统/设备类别
- 通过应用内编辑器创建用户自定义技能
- 技能编辑和删除(仅限自定义)
- 技能导出为 `SKILL.md` 文件
- 从 `SKILL.md` 文件导入技能并验证
- 参数定义和在技能提示词中的替换
- 技能管理界面,用于查看和管理所有技能

### 不包含的功能 (V1)
- 技能市场或在线技能商店
- 技能版本控制/版本历史(仅存储当前版本)
- 技能链式调用(一个技能触发另一个技能)
- 通用参数输入表单之外的技能专用 UI
- 技能提示词中的条件逻辑(不支持 if/else、循环 -- 仅线性提示词指令)
- 技能执行审批流程(技能使用现有工具权限)
- 技能分析仪表板(基本使用计数为可选)
- 技能调度/自动化(如"每天运行此技能")
- 加密或 DRM 保护的技能
- 技能云同步(仅本地;通过导出/导入分享)

## 业务规则

### 技能规则
1. 每个技能必须有唯一名称(在创建和导入时强制)
2. 技能名称遵循 snake-case 约定(如 `summarize-file`)
3. 技能名称必须为 2-50 个字符,仅包含小写字母、数字和连字符
4. 显示名称必须非空
5. 描述必须非空
6. 提示词内容必须非空
7. 内置技能只读,不能删除
8. 用户自定义技能可以自由创建、编辑和删除
9. 技能是全局资源,独立于 Agent

### 参数规则
1. 参数名称遵循 snake_case 约定
2. V1 支持的参数类型: `string`(仅此一种,后续可添加更多)
3. 必填参数必须在技能执行前提供
4. 提示词内容中的参数占位符使用 `{{parameter_name}}` 语法
5. 如果必填参数未提供,AI 应向用户询问

### 触发规则
1. `/` 命令触发仅在 `/` 是输入的第一个字符时激活
2. 自动完成列表同时根据 `name` 和 `display_name` 过滤
3. 如果技能没有必填参数,立即触发
4. 如果技能有必填参数,触发前显示参数输入表单
5. AI 可以通过 `load_skill` 工具自主触发技能,无需用户干预

### 导入/导出规则
1. 导出文件命名为 `SKILL.md`
2. 导入验证: 文件是有效 Markdown、有有效 YAML frontmatter、包含必填字段(name、display_name、description、提示词内容)
3. 导入时,系统根据文件中的 `name` 字段创建对应目录,并将 `SKILL.md` 存入其中
4. 如果导入时已存在同名技能(即目录已存在),提示用户选择:重命名或替换
5. 导入的技能始终存储为用户自定义技能(即使从内置技能导出)

## 非功能性需求

### 性能要求
- 技能注册表生成(用于系统提示词)< 50ms 完成
- `load_skill` 工具执行 < 100ms 完成(本地文件读取)
- / 命令自动完成列表 < 100ms 出现
- 技能导入验证 < 200ms 完成
- 技能创建/保存 < 200ms 完成

### 数据要求
- 技能文件使用 UTF-8 编码
- 最大技能文件大小: 100KB(提示词内容通常 1-10KB)
- 用户自定义技能数量无限制(实际受存储空间限制)

### 安全要求
- 技能提示词内容是纯文本,不能直接执行代码
- 导入的技能应被视为不可信任的 -- 它们只能使用现有工具引导 AI 行为,而这些工具已有自己的权限检查
- 技能文件不包含任何密钥或 API 密钥
- `load_skill` 工具仅从指定的技能目录读取(不能读取任意文件)

## 错误处理

### 可能的错误场景

1. **技能未找到**
   - 原因: `load_skill` 传入的名称不匹配任何技能
   - 处理: 向 AI 返回错误: "Skill [name] not found. Available skills: [list]"
   - AI 可以向用户推荐可用的技能

2. **导入文件无效**
   - 原因: 导入的文件不是有效 Markdown、缺少 frontmatter 或缺少必填字段
   - 处理: 显示错误对话框: "无效的技能文件: [具体原因]"
   - 用户可以修复文件后重试

3. **技能需要不可用的工具**
   - 原因: 技能的 `tools_required` 包含不在当前 Agent 工具集中的工具
   - 处理: `load_skill` 仍然成功(技能被加载),但包含警告: "注意: 此技能需要 [tool_name],该工具对当前 Agent 不可用"
   - AI 应告知用户此限制

4. **参数占位符未解析**
   - 原因: 技能提示词包含 `{{param}}` 但参数未提供
   - 处理: AI 应检测到未解析的占位符,向用户询问缺失的值

5. **导入时名称冲突**
   - 原因: 导入的技能与已存在的技能同名
   - 处理: 提示用户: "名为 [name] 的技能已存在。重命名还是替换?"

6. **技能文件读取错误**
   - 原因: 读取技能文件时的文件系统错误
   - 处理: 向 AI 返回错误: "Failed to load skill [name]: [error details]"

- 验证 `load_skill` 对不存在的技能返回错误
- 验证系统提示词包含技能注册列表
- 验证输入框中的 `/` 命令显示技能自动完成
- 验证自动完成按名称和显示名称过滤
- 验证选择无必填参数的技能时立即触发
- 验证选择有必填参数的技能时显示参数表单
- 验证参数值在提示词内容中被替换
- 验证 UI 技能按钮打开技能选择底部弹出表
- 验证 AI 可以自主调用 `load_skill`
- 验证自定义技能创建正确保存
- 验证自定义技能编辑正确更新
- 验证自定义技能删除从列表和 / 命令中移除
- 验证内置技能不能被编辑或删除
- 验证技能导出生成有效的 `SKILL.md` 文件
- 验证技能导入正确解析和存储
- 验证导入拒绝无效文件并给出清晰错误
- 验证导入处理名称冲突(重命名/替换)
- 验证技能无论哪个 Agent 活跃都能工作

### 边界条件
- 参数列表为空的技能(不需要参数)
- 提示词内容接近最大大小(接近 100KB)的技能
- 导入从不同 app 版本导出的技能
- 创建名称非常相似的两个技能
- 在对话中使用技能时删除该技能
- 导入需要未实现工具的技能
- 没有技能可用时的 `/` 命令
- 非常长的参数值
- 没有 `{{}}` 占位符的技能提示词(不需要参数替换)
- 快速连续触发技能
