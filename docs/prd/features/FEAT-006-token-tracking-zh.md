## 用户故事

**作为** 一个使用自己 API key 的用户，
**我希望** 能看到每条消息、每个会话和每个模型消耗了多少 token，
**以便** 我能了解自己的使用模式并管理 API 开支。

### 典型场景

1. 用户发送一条消息，希望看到该次交互消耗了多少输入/输出 token。
2. 用户打开一个会话，查看该对话的总 token 用量。
3. 用户打开"用量统计"界面，查看过去一周按模型分列的用量明细。
4. 用户比较不同模型的 token 消耗量，以决定哪个模型更高效。

## 功能描述

### 概述

Token 用量追踪提供三个层级的 API token 消耗可见性：按消息、按会话和全局。所有计数均来自 provider API 在 streaming 响应中返回的 token 用量数据。V1 版本仅追踪 token 数量，不进行费用估算或定价。

### 展示层级

#### 1. 按消息 Token 计数

每条 AI 回复消息内联展示其 token 用量：
- **Input tokens**：发送给模型的请求中的 token 数量（prompt + context）
- **Output tokens**：模型回复中的 token 数量

以小标签形式显示在 AI 消息下方，紧挨已有的模型 ID 标签。示例：`claude-sonnet-4-20250514 | 1,234 in / 567 out`

用户消息不显示 token 计数（用户消息没有 API 调用）。

#### 2. 按会话 Token 汇总

每个会话显示累计 token 用量：
- 会话的总输入 token 数
- 会话的总输出 token 数
- 在会话抽屉中可见（位于会话标题/预览下方），也可在会话详情/信息视图中查看

#### 3. 全局用量统计界面

一个可从"设置"进入的专用界面（"用量统计"入口）：

- **时间段选择器**：今天 / 本周 / 本月 / 全部时间
- **按模型分列明细**：每个使用过的模型展示：
  - 模型名称/ID
  - 总输入 token 数
  - 总输出 token 数
  - 总 token 数（输入 + 输出）
  - 消息数量
- **合计行**：所选时间段内所有模型的总和
- 数据从本地存储的消息记录中读取（无需外部 API）

### 数据来源

Token 计数来自 streaming 过程中发出的 `StreamEvent.Usage` 事件，这些数据已被捕获并存储在 `Message.tokenCountInput` 和 `Message.tokenCountOutput` 字段中。全局统计界面从 messages 表中聚合这些字段。

注意：并非所有 provider 都会在每个 streaming 响应中返回 token 用量。当某条消息的 token 数据不可用时，存储为 `null`，并从聚合中排除（不视为零）。

## 验收标准

必须通过（全部为必需项）：
- [ ] 每条 AI 回复消息在下方显示输入/输出 token 计数
- [ ] 仅在 provider 返回数据时显示 token 计数（不显示虚假的零值）
- [ ] 会话抽屉中显示每个会话的累计输入/输出 token 计数
- [ ] 设置界面中有"用量统计"入口
- [ ] 用量统计界面按模型显示 token 明细
- [ ] 用量统计界面支持时间段选择：今天 / 本周 / 本月 / 全部时间
- [ ] 每个模型行显示：模型名称、输入 token 数、输出 token 数、总 token 数、消息数
- [ ] 底部有合计行，显示所有模型的总和
- [ ] Token 计数使用逗号分隔的数字格式（例如 1,234,567）

可选（锦上添花）：
- [ ] 按模型用量的柱状图或可视化展示
- [ ] 趋势对比（本周与上周）

## UI/UX 要求

### 按消息展示

在每条 AI 回复下方，扩展现有标签行：

```
[Copy] [Regenerate]  claude-sonnet-4-20250514 | 1,234 in / 567 out
```

Token 标签使用 `labelSmall` 字体排版，`onSurfaceVariant` 颜色，与模型 ID 样式一致。

### 会话抽屉

在每个会话的预览文字下方，添加一个小型 token 汇总：

```
Session Title
Last message preview text...
2 hours ago                        12.3K tokens
```

使用 `labelSmall`，右对齐。显示总 token 数（输入 + 输出合计），大数字使用 K/M 缩写。

### 用量统计界面

```
Usage Statistics
[Today] [This Week] [This Month] [All Time]    <-- chip row

Model                  Input      Output     Total    Messages
--------------------------------------------------------------
claude-sonnet-4-...    45,230     12,450    57,680      42
gpt-4o                 23,100      8,300    31,400      28
gemini-2.0-flash        5,600      2,100     7,700      15
--------------------------------------------------------------
Total                  73,930     22,850    96,780      85
```

- 顶栏：标题"用量统计"配返回箭头
- Chip 行用于时间段选择（单选）
- 表格/列表展示模型行
- 每行为一个 `Surface` 卡片或列表项
- 底部合计行使用粗体文字

## 功能边界

### 包含
- 按消息 token 展示（输入/输出）
- 会话抽屉中的按会话 token 汇总
- 全局用量统计界面，含时间段过滤和模型明细
- 带分隔符的数字格式

### 不包含
- 费用/价格估算
- 预算提醒或消费限额
- 按 agent 分列的用量明细
- 数据导出（CSV 等）
- 历史趋势图表
- 未发送消息的 token 预测或估算

## 业务规则

1. Token 计数完全来源于 provider API 响应。应用不独立估算或计算 token。
2. 当 provider 未返回 token 用量数据时，消息的 token 字段为 `null`。这些消息从聚合总计中排除。
3. Token 计数按消息存储，会话和全局视图实时聚合。
4. 时间段过滤器使用消息的 `createdAt` 时间戳。
5. "今天"指从本地时间午夜开始。"本周"指从本地时间周一 00:00 开始。"本月"指从当月 1 日开始。

## 测试要点

### 功能测试 -- 用户操作应用

#### 测试 1：按消息 token 展示

**前置条件**：已配置一个返回 token 用量的 provider 和模型。

| 步骤 | 用户操作 | 预期结果 |
|------|----------|----------|
| 1 | 发送消息 "Hello" | AI 回复。 |
| 2 | 观察 AI 回复下方 | 模型 ID 标签可见。旁边显示 token 计数，如 "1,234 in / 567 out"。 |
| 3 | 发送另一条消息 | 第二条 AI 回复同样显示 token 计数。 |

#### 测试 2：Provider 未返回用量时的 token 展示

**前置条件**：使用一个在 streaming 中不返回 token 计数的 provider/模型。

| 步骤 | 用户操作 | 预期结果 |
|------|----------|----------|
| 1 | 发送一条消息 | AI 回复。 |
| 2 | 观察 AI 回复下方 | 显示模型 ID 但不显示 token 计数标签（不显示 "0 in / 0 out"）。 |

#### 测试 3：会话抽屉中的会话 token 汇总

| 步骤 | 用户操作 | 预期结果 |
|------|----------|----------|
| 1 | 进行包含多条消息的对话 | 收到多条 AI 回复。 |
| 2 | 打开会话抽屉（汉堡图标） | 当前会话在预览下方右对齐显示总 token 数（如 "12.3K tokens"）。 |

#### 测试 4：用量统计界面

| 步骤 | 用户操作 | 预期结果 |
|------|----------|----------|
| 1 | 进入设置 | "用量统计"入口可见。 |
| 2 | 点击"用量统计" | 统计界面打开。默认时间段为"全部时间"。 |
| 3 | 查看模型明细 | 每个使用过的模型显示输入/输出/总 token 数和消息数。底部有合计行。 |
| 4 | 点击"今天" chip | 表格更新为仅显示今天的用量。如果今天没有消息，表格为空或显示零。 |
| 5 | 点击"本周" chip | 表格更新为显示本周的用量。 |

#### 测试 5：数字格式

| 步骤 | 用户操作 | 预期结果 |
|------|----------|----------|
| 1 | 进行足够多的对话以积累大量 token 计数 | Token 计数以逗号分隔显示（如 "1,234,567" 而非 "1234567"）。 |

### 边界情况

- 新安装无任何消息：用量统计界面显示空状态
- 仅包含用户消息的会话（所有 AI 回复均失败）：会话 token 计数为 0 或不显示
- 仅使用过一次的模型：仍出现在明细中
- 非常大的数字（数百万 token）：验证格式和布局不会错乱
